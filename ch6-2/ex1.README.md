# 📘 6장-2 실습과제 1 — 디폴트 매개변수는 왜 선언부에 써야 하는가?

---

## 📌 문제
> 디폴트 매개변수는 함수의 **선언**에서 정의해주어야 한다.
> 이유를 설명하라. 만약 함수의 **정의**에서 해준다면 어떻게 될지 생각해보라.

---

## ✅ 답

### 1. 컴파일러가 함수를 처리하는 순서

C++ 컴파일러는 소스코드를 **위에서 아래로** 한 줄씩 읽으며 처리한다.
함수를 호출하는 코드(`main()` 등)를 만나는 순간, 컴파일러는 그 함수에 대한
정보가 이미 위쪽에 있어야 호출 형태가 올바른지 판단할 수 있다.

```cpp
void star(int a = 5); // ← 선언부: 컴파일러가 여기서 디폴트값을 알게 됨

int main() {
    star();   // 컴파일러: "a의 디폴트값이 5구나" → star(5)로 변환 ✅
}

void star(int a) { // ← 정의부: 디폴트값 없음
    for (int i = 0; i < a; i++) cout << '*';
}
```

---

### 2. 선언부에 디폴트값을 써야 하는 이유

컴파일러가 `star();` 를 만나는 시점에는 아직 함수 **정의부**를 읽지 않은 상태다.
따라서 **선언부에 디폴트값이 없으면**, 컴파일러는 인자가 없는 `star()` 호출이
올바른지 판단할 수 없어 **컴파일 오류**가 발생한다.

> 핵심: 컴파일러는 **함수 호출 시점**에 디폴트값을 알아야 하고,
> 그 시점에 이미 읽혀 있는 것은 **선언부**이므로, 디폴트값은 선언부에 써야 한다.

---

### 3. 정의부에만 디폴트값을 쓰면 어떻게 되는가?

```cpp
// 선언부 — 디폴트값 없음
void star(int a);

int main() {
    star(); // ❌ 컴파일 오류: a에 전달할 인자가 없음
            //    컴파일러는 선언부만 보고 판단하므로 디폴트값을 모름
}

// 정의부 — 디폴트값 있음 (하지만 이미 늦음)
void star(int a = 5) {
    for (int i = 0; i < a; i++) cout << '*';
}
```

> 컴파일러가 `main()` 을 처리할 때 아직 정의부를 읽지 않았으므로,
> 정의부의 디폴트값은 **의미가 없다**.
> 결과적으로 `star()` 호출은 인자 부족으로 **컴파일 오류**가 발생한다.

---

### 4. 예외: 선언부 없이 정의부가 먼저 나오는 경우

정의부가 호출 코드보다 **위에** 있다면, 정의부에 디폴트값을 써도 동작한다.

```cpp
// 선언부 없이 정의부가 먼저 등장
void star(int a = 5) {    // ✅ 정의부에 디폴트값 → 호출 전에 이미 읽힘
    for (int i = 0; i < a; i++) cout << '*';
}

int main() {
    star(); // ✅ 정상 동작: 컴파일러가 이미 디폴트값을 알고 있음
}
```

> 하지만 이 방식은 함수가 많아지면 구조가 복잡해지므로,
> 일반적으로 **선언부와 정의부를 분리**하고 **선언부에 디폴트값**을 쓰는 것이 표준 방식이다.

---

## 📝 핵심 요약

| 상황 | 결과 |
|------|------|
| 선언부에 디폴트값, 정의부에는 없음 | ✅ 정상 동작 (일반적인 방식) |
| 선언부 없이 정의부가 호출보다 위에 있고 디폴트값 포함 | ✅ 정상 동작 |
| 선언부에 디폴트값 없고, 정의부에만 디폴트값 | ❌ 컴파일 오류 |
| 선언부와 정의부 **둘 다**에 디폴트값 | ❌ 컴파일 오류 (중복 지정 불가) |

> **결론**: 디폴트값은 컴파일러가 함수 호출을 처리하는 시점 이전에 알려져 있어야 하므로,
> 선언부와 정의부를 분리하는 경우 반드시 **선언부**에 디폴트값을 지정해야 한다.
