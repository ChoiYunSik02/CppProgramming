# C++ 4-1장 실습과제5번 - 소스코드 설명

---

## 과제 목표

실습과제4의 `Triangle` 객체 배열을 **객체 포인터를 이용한 코드로 변경**한다.  
강의노트 23페이지의 **4가지 방법**을 모두 사용하여 배열 원소에 접근한다.

---

## 실습과제4와의 차이점

| 항목 | 실습과제4 | 실습과제5 |
|------|-----------|-----------|
| 배열 원소 접근 방식 | `triArray[i].getArea()` | 포인터 4가지 방법 사용 |
| 객체 포인터 사용 | 없음 | `Triangle* p = triArray` |
| 생성/소멸 메시지 | 출력 있음 | 출력 없음 (불필요하여 제거) |

---

## 클래스 설명 : `Triangle`

### 멤버 변수

| 변수명 | 자료형 | 접근 지정자 | 설명 |
|--------|--------|-------------|------|
| `width` | `int` | `private` | 삼각형의 밑변 |
| `height` | `int` | `private` | 삼각형의 높이 |

### 멤버 함수

#### 기본 생성자 `Triangle()`
```cpp
Triangle() { width = 1; height = 1; }
```
- 매개변수 없이 호출되는 생성자, `width=1`, `height=1` 로 초기화

#### 매개변수 생성자 `Triangle(int a, int b)`
```cpp
Triangle(int a, int b) { width = a; height = b; }
```
- 밑변과 높이를 인자로 받아 초기화
- 실습과제4와 달리 `cout` 생성 메시지 제거

#### 소멸자 `~Triangle()`
```cpp
~Triangle() {}
```
- 실습과제4와 달리 `cout` 소멸 메시지 제거 (빈 소멸자)

#### `getArea()`
```cpp
double getArea() {
    return (1.0 / 2.0) * width * height;
}
```
- 삼각형 넓이 `(밑변 × 높이) / 2` 계산 후 반환

---

## main 함수 설명

### 객체 배열 및 포인터 선언

```cpp
Triangle triArray[3] = { Triangle(2,2), Triangle(4,4), Triangle(6,6) };
Triangle* p = triArray;
```

- 원소 3개짜리 객체 배열 선언과 동시에 각 원소 초기화
- `p = triArray` : 포인터 `p` 에 배열의 첫 번째 원소 주소 대입
- `triArray` 는 `&triArray[0]` 과 동일한 주소를 가리킴

| 원소 | width | height | getArea() |
|------|-------|--------|-----------|
| `triArray[0]` | 2 | 2 | 2.0 |
| `triArray[1]` | 4 | 4 | 8.0 |
| `triArray[2]` | 6 | 6 | 18.0 |

---

### 4가지 포인터 접근 방법

#### 방법 1 : `(배열명+i)->멤버함수()`

```cpp
for (int i = 0; i < 3; i++)
    cout << "삼각형" << i << "의 면적은 " << (triArray+i)->getArea() << endl;
```

- `triArray` 는 포인터 상수로, `+i` 로 주소를 이동하여 각 원소에 접근
- `(triArray+0)` = `triArray[0]` 의 주소
- `(triArray+1)` = `triArray[1]` 의 주소
- `(triArray+2)` = `triArray[2]` 의 주소

---

#### 방법 2 : `(포인터+i)->멤버함수()`

```cpp
for (int i = 0; i < 3; i++)
    cout << "삼각형" << i << "의 면적은 " << (p+i)->getArea() << endl;
```

- 방법 1과 동일하지만 배열명 대신 포인터 변수 `p` 를 사용
- `p` 는 `triArray` 와 같은 주소를 가리키므로 결과 동일

---

#### 방법 3 : `포인터[i].멤버함수()`

```cpp
for (int i = 0; i < 3; i++)
    cout << "삼각형" << i << "의 면적은 " << p[i].getArea() << endl;
```

- 포인터를 배열처럼 인덱스로 접근
- `p[i]` 는 `*(p+i)` 와 동일한 표현

---

#### 방법 4 : `p->멤버함수(); p++`

```cpp
p = triArray; // 포인터 초기화 (방법3 이후 p가 이동했을 경우 대비)
for (int i = 0; i < 3; i++) {
    cout << "삼각형" << i << "의 면적은 " << p->getArea() << endl;
    p++;
}
```

- `p++` 로 포인터를 한 칸씩 직접 이동하며 순회
- 루프 시작 전 `p = triArray` 로 **반드시 포인터를 초기화** 해야 함
- `p++` 는 `Triangle` 타입의 크기만큼 주소가 증가함

> ⚠️ 방법 4에서 `p = triArray` 초기화를 빠뜨리면, 이전 방법에서 `p` 가 이동된 상태로  
> 잘못된 메모리를 가리켜 **예상치 못한 결과** 가 발생할 수 있다.

---

## 4가지 방법 비교 요약

| 방법 | 코드 형태 | 포인터 이동 여부 | 특징 |
|------|-----------|----------------|------|
| 방법 1 | `(triArray+i)->` | 없음 | 배열명을 포인터처럼 사용 |
| 방법 2 | `(p+i)->` | 없음 | 포인터 변수로 산술 연산 |
| 방법 3 | `p[i].` | 없음 | 포인터를 배열 인덱스로 접근 |
| 방법 4 | `p->` + `p++` | 있음 (p 직접 이동) | 루프마다 포인터 증가, 초기화 필요 |

---

## 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| `p = triArray` | 배열의 첫 번째 원소 주소를 포인터에 대입 |
| `(p+i)->멤버` | 포인터 산술로 i번째 원소 접근 |
| `p[i].멤버` | 포인터를 배열처럼 인덱스로 접근 |
| `p++` | 포인터를 다음 원소로 이동 (sizeof(Triangle) 만큼 주소 증가) |
