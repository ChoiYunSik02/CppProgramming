# C++ 4-1장 실습과제 5번 - 소스코드 설명

 이름 : 최윤식

---

## 과제 목표

실습과제4의 `Triangle` 객체 배열을 **객체 포인터를 이용한 코드로 변경**한다.  
강의노트 23페이지의 **4가지 방법**을 모두 사용하여 배열 원소에 접근한다.

---

## 클래스 설명 : `Triangle`

### 멤버 변수

| 변수명 | 자료형 | 접근 지정자 | 설명 |
|--------|--------|-------------|------|
| `width` | `int` | `private` | 삼각형의 밑변 |
| `height` | `int` | `private` | 삼각형의 높이 |

### 멤버 함수

#### 기본 생성자 `Triangle()`
```cpp
Triangle() { width = 1; height = 1; }
```
- 매개변수 없이 호출되는 생성자, `width=1`, `height=1` 로 초기화

#### 매개변수 생성자 `Triangle(int a, int b)`
```cpp
Triangle(int a, int b) { width = a; height = b; }
```
- 밑변과 높이를 인자로 받아 초기화

#### 소멸자 `~Triangle()`
```cpp
~Triangle() {}
```
- 빈 소멸자 (생성/소멸 메시지 없음)

#### `getArea()`
```cpp
double getArea() {
    return (1.0 / 2.0) * width * height;
}
```
- 삼각형 넓이 `(밑변 × 높이) / 2` 계산 후 반환
- `1.0 / 2.0` 으로 작성하여 정수 나눗셈 오류 방지

## main 함수 설명

### 객체 배열 및 포인터 선언

```cpp
Triangle tri[3] = { Triangle(2, 2), Triangle(4, 4), Triangle(6, 6) };
Triangle* p = tri;
```

- 원소 3개짜리 객체 배열 `tri` 선언과 동시에 초기화
- `p = tri` : 포인터 `p` 에 배열 첫 번째 원소의 주소 대입
- `tri` 는 `&tri[0]` 과 동일한 주소를 가리키는 포인터 상수

| 원소 | width | height | getArea() |
|------|-------|--------|-----------|
| `tri[0]` | 2 | 2 | 2.0 |
| `tri[1]` | 4 | 4 | 8.0 |
| `tri[2]` | 6 | 6 | 18.0 |

---

### 방법 1 : `(배열명+i)->멤버함수()`

```cpp
for (int i = 0; i < 3; i++) {
    cout << "삼각형" << i << "의 면적은 " << (tri + i)->getArea() << endl;
}
```

- `tri` 는 배열의 첫 번째 원소 주소를 가리키는 포인터 상수
- `tri + i` 로 i번째 원소의 주소를 계산하여 `->` 로 멤버 접근
- 배열명을 직접 포인터처럼 활용하는 방법

| i | tri+i | 가리키는 원소 |
|---|-------|-------------|
| 0 | tri+0 | tri[0] |
| 1 | tri+1 | tri[1] |
| 2 | tri+2 | tri[2] |

---

### 방법 2 : `(포인터+i)->멤버함수()`

```cpp
for (int i = 0; i < 3; i++) {
    cout << "삼각형" << i << "의 면적은 " << (p + i)->getArea() << endl;
}
```

- 방법 1과 동일하지만 배열명 `tri` 대신 포인터 변수 `p` 를 사용
- `p` 는 `tri` 와 같은 주소를 가리키므로 결과 동일
- `p` 자체는 이동하지 않고 `p+i` 로 임시 주소만 계산

---

### 방법 3 : `포인터[i].멤버함수()`

```cpp
for (int i = 0; i < 3; i++) {
    cout << "삼각형" << i << "의 면적은 " << p[i].getArea() << endl;
}
```

- 포인터를 배열 인덱스로 접근하는 방법
- `p[i]` 는 `*(p+i)` 와 완전히 동일한 표현
- `->` 대신 `.` 연산자로 멤버 접근

---

### 방법 4 : `p->멤버함수(); p++`

```cpp
p = tri; // 포인터 초기화
for (int i = 0; i < 3; i++) {
    cout << "삼각형" << i << "의 면적은 " << p->getArea() << endl;
    p++;
}
```

- `p++` 로 포인터를 한 칸씩 직접 이동하며 순회
- 루프 시작 전 `p = tri` 로 포인터를 처음 위치로 **반드시 초기화**
- `p++` 는 `Triangle` 타입의 크기만큼 주소가 증가함

> ⚠️ `p = tri` 초기화를 빠뜨리면 방법 3 이후 `p` 가 배열 범위를 벗어나  
> 잘못된 메모리를 가리키게 되어 **예상치 못한 결과** 가 발생한다.

---

## 4가지 방법 비교 요약

| 방법 | 코드 형태 | p 이동 여부 | 특징 |
|------|-----------|------------|------|
| 방법 1 | `(tri+i)->getArea()` | 없음 | 배열명을 포인터 상수로 활용 |
| 방법 2 | `(p+i)->getArea()` | 없음 | 포인터 변수로 산술 연산 |
| 방법 3 | `p[i].getArea()` | 없음 | 포인터를 배열 인덱스로 접근 |
| 방법 4 | `p->getArea(); p++` | 있음 | 루프마다 포인터 직접 증가, 초기화 필요 |

---

## 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| `p = tri` | 배열 첫 번째 원소의 주소를 포인터에 대입 |
| `(tri+i)->멤버` | 배열명으로 포인터 산술 연산 후 접근 |
| `(p+i)->멤버` | 포인터 변수로 포인터 산술 연산 후 접근 |
| `p[i].멤버` | 포인터를 배열 인덱스처럼 사용, `*(p+i)` 와 동일 |
| `p->멤버; p++` | 포인터를 직접 이동하며 순차 접근 |
