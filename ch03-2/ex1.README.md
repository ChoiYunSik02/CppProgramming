## 1. 생성자를 통해 멤버변수를 초기화하는 이유

### 초기화를 안했을때

객체를 생성했을 때 멤버변수에는 **이전 메모리에 남아있던 쓰레기값**이 들어가 있다.
즉 이 상태로 면적을 계산하면 전혀 엉뚱한 결과가 나오게 된다.

### 생성자로 초기화

**생성자는 객체가 만들어지는 순간 자동으로 호출**되기 때문에,
개발자가 따로 초기화 코드를 작성하지 않아도 안전하게 초기값이 설정된다.

---

### 생성자로 초기화해야 하는 이유 정리

| 이유 | 설명 |
|---|---|
| 안전성 | 쓰레기값으로 인한 버그 방지 |
| 자동 호출 | 객체 생성 시 자동 실행, 초기화를 빠뜨릴 가능성 없음 |
| 일관성 | 어디서 객체를 만들어도 항상 같은 초기 상태 보장 |
| const/참조 멤버 | `const`나 `&` 멤버변수는 **반드시 생성자에서만 초기화 가능** |

---

## 2. 생성자 몸체보다 멤버 초기화 리스트를 사용하는 이유

### 두 가지 방법 비교

**방법 1 - 생성자 몸체에서 초기화**
```cpp
class Rectangle {
    int width;
    int height;
public:
    Rectangle(int w, int h) {
        width = w;   // 이건 초기화가 아니라 "대입"
        height = h;  // 멤버변수가 먼저 생성된 후 값을 덮어씀
    }
};
```

**방법 2 - 멤버 초기화 리스트**
```cpp
class Rectangle {
    int width;
    int height;
public:
    Rectangle(int w, int h) : width(w), height(h) {
        // 멤버변수가 생성되는 순간 바로 초기화됨
    }
};
```

---

### 핵심 차이: 초기화 vs 대입

| 구분 | 생성자 몸체 | 멤버 초기화 리스트 |
|---|---|---|
| 동작 방식 | 생성 → 대입 (2단계) | 생성과 동시에 초기화 (1단계) |
| 성능 | 상대적으로 비효율적 | 더 효율적 |
| const 멤버 | ❌ 사용 불가 | ✅ 사용 가능 |
| 참조(&) 멤버 | ❌ 사용 불가 | ✅ 사용 가능 |
| 객체 멤버 | 기본 생성자 먼저 호출됨 | 원하는 생성자 바로 호출 |

---

### const 멤버는 반드시 초기화 리스트 사용

```cpp
class Circle {
    const double PI;  // const 멤버는 선언 후 값 변경 불가
public:
    // ❌ 몸체에서 대입 → 컴파일 에러!
    Circle() {
        PI = 3.14;  // 에러: const 변수에 대입 불가
    }

    // ✅ 초기화 리스트 → 정상 동작
    Circle() : PI(3.14) { }
};
```

---

### 결론

> 멤버 초기화 리스트는 **"처음부터 올바른 값으로 태어나게"** 하는 방식이고,
> 몸체에서 대입은 **"일단 태어난 후 값을 바꾸는"** 방식이에요.
> 가능하면 항상 **멤버 초기화 리스트를 사용하는 것이 권장**됩니다.

## 3. 위임 생성자와 타겟 생성자로 나누어 작성하는 이유

### 개념 설명

```cpp
class Rectangle {
    int x, y, width, height;
public:
    // 타겟 생성자: 실제 초기화 작업을 담당
    Rectangle(int x, int y, int w, int h) : x(x), y(y), width(w), height(h) {
        cout << "초기화 완료" << endl;
    }

    // 위임 생성자: 타겟 생성자에게 초기화를 위임
    Rectangle() : Rectangle(0, 0, 0, 0) { }           // 기본값으로 위임
    Rectangle(int w, int h) : Rectangle(0, 0, w, h) { } // 좌표만 기본값으로 위임
};
```

---

### 위임 생성자를 사용하지 않으면?

```cpp
// ❌ 코드 중복 발생
class Rectangle {
    int x, y, width, height;
public:
    Rectangle() {
        x = 0; y = 0; width = 0; height = 0;  // 중복!
        cout << "초기화 완료" << endl;          // 중복!
    }
    Rectangle(int w, int h) {
        x = 0; y = 0; width = w; height = h;  // 중복!
        cout << "초기화 완료" << endl;          // 중복!
    }
    Rectangle(int x, int y, int w, int h) {
        x = x; y = y; width = w; height = h;
        cout << "초기화 완료" << endl;
    }
};
```

초기화 로직이 바뀌면 **모든 생성자를 일일이 수정**해야 해요.

---

### 위임 생성자를 사용하면?

```
Rectangle()          →  Rectangle(0, 0, 0, 0)  ← 실제 초기화는 여기서만!
Rectangle(w, h)      →  Rectangle(0, 0, w, h)  ← 여기서도 위임
Rectangle(x,y,w,h)  →  직접 초기화 (타겟)
```

초기화 로직이 **타겟 생성자 한 곳에만** 있으므로 유지보수가 쉬워요.

---

### 나누어 작성하는 이유 정리

| 이유 | 설명 |
|---|---|
| 코드 중복 제거 | 초기화 로직을 타겟 생성자 한 곳에 집중 |
| 유지보수 편의 | 로직 변경 시 타겟 생성자만 수정하면 됨 |
| 가독성 향상 | 각 생성자의 역할이 명확히 분리됨 |
| 버그 방지 | 여러 곳에 중복된 코드에서 생기는 실수 방지 |

## 4. 함수 몸체에서 생성자를 직접 호출하면?

### 직접 호출해보기

```cpp
class Rectangle {
public:
    int width, height;
    Rectangle(int w, int h) : width(w), height(h) {
        cout << "생성자 호출됨" << endl;
    }
};

int main() {
    Rectangle rect(5, 3);

    // 함수 몸체에서 생성자 직접 호출 시도
    rect.Rectangle(10, 20);  // ❌ 컴파일 에러!
}
```

---

### 무슨 일이 일어나는가?

```cpp
int main() {
    Rectangle rect(5, 3);

    // 아래처럼 쓰면 컴파일은 되지만...
    rect = Rectangle(10, 20);  // 새 임시 객체를 만들어서 대입
}
```

| 상황 | 결과 |
|---|---|
| `rect.Rectangle(10, 20)` | ❌ **컴파일 에러** — 생성자는 직접 호출 불가 |
| `rect = Rectangle(10, 20)` | ⚠️ **임시 객체 생성 후 대입** — 원래 객체를 초기화하는 게 아님 |

---

### 왜 직접 호출이 불가능한가?

생성자는 **객체가 처음 만들어질 때 딱 한 번만 자동으로 호출**되도록 설계된 특별한 함수예요.

```
일반 함수  → 언제든 원하는 만큼 호출 가능
생성자     → 객체 생성 시 딱 1번, 자동 호출만 가능
```

이미 만들어진 객체에서 생성자를 다시 호출하는 건 **"이미 태어난 사람을 다시 태어나게 하는 것"** 과 같아서 허용되지 않아요.

---

### 재초기화가 필요하다면?

```cpp
class Rectangle {
public:
    int width, height;
    Rectangle(int w, int h) : width(w), height(h) { }

    // 별도의 reset 함수를 만들어서 사용
    void reset(int w, int h) {
        width = w;
        height = h;
    }
};

int main() {
    Rectangle rect(5, 3);
    rect.reset(10, 20);  // ✅ 이렇게 해야 함
}
```

> 생성자를 다시 호출하고 싶은 상황이라면, 별도의 **초기화 멤버함수(`reset`, `init` 등)** 를 만들어 사용하는 것이 올바른 방법이에요.

| 생성자로 초기화하는 이유 | 쓰레기값 방지, 객체 생성 시 자동 호출로 안전성 보장 |
| 멤버 초기화 리스트 권장 이유 | 초기화(1단계) vs 대입(2단계), const/참조 멤버 초기화 가능 |
