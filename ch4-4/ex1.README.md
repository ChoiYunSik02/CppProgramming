## 1. string 객체가 크기를 자동 조절하는 동작 원리

### 핵심 개념: 동적 메모리 + 용량 자동 확장 (Capacity Doubling)

`std::string`은 내부적으로 **힙(heap) 메모리**에 문자 배열을 동적으로 할당하여 관리합니다.  
`char arr[100]`처럼 스택에 고정 크기로 잡는 C-스트링과 달리, 문자열이 길어지면 알아서 더 큰 메모리 공간을 확보합니다.

### 내부 구조 (개념도)

```
string 객체 (스택)
┌────────────────────────┐
│  char* data   ─────────┼──→ [ h | e | l | l | o | \0 | _ | _ ]  (힙)
│  size_t size  = 5      │     ↑ 실제 문자열 길이: 5
│  size_t capacity = 8   │     ↑ 할당된 버퍼 크기: 8 (여유 공간 포함)
└────────────────────────┘
```

### 용량 확장 방식: 2배 증가 전략 (Doubling Strategy)

문자열을 추가(`+=`, `append()`)하다가 현재 `capacity`가 부족해지면:

1. 기존 `capacity`의 **약 2배**로 새 메모리를 힙에 할당
2. 기존 내용을 새 메모리로 복사
3. 기존 메모리는 `delete[]`로 해제

```
"hello" 상태:  size=5, capacity=8
+= " world" 시도 → 필요 size=11 > capacity=8
→ 새 capacity = 16으로 재할당
→ "hello world" 저장 완료
```

> 이 전략 덕분에 매번 재할당하지 않아도 되므로 성능 효율적입니다.

### char 배열과 비교

| 항목 | `char arr[N]` | `string` |
|------|---------------|----------|
| 크기 | 컴파일 타임 고정 | 런타임 자동 확장 |
| 메모리 위치 | 스택 | 힙 (내부 버퍼) |
| 초과 입력 | 버퍼 오버플로우 위험 | 자동 재할당으로 안전 |
| 크기 확인 | `strlen()` | `.length()` / `.size()` |

→ **앞으로 C++에서는 `char` 배열 대신 `string` 클래스를 사용합니다.**

## 2. 엔터키와 getline의 구분문자(delimiter) 차이

### 엔터키 (`\n`)의 역할: OS 수준의 입력 완료 신호

사용자가 키보드에서 **엔터키**를 누르면:
- 운영체제(OS)가 입력된 모든 문자 + `\n`을 **표준 입력 스트림 버퍼**에 넣습니다.
- 이 시점부터 `cin`/`getline` 등의 함수가 버퍼에서 데이터를 읽을 수 있게 됩니다.
- 엔터키 자체는 `cin >>` 방식에서는 **건너뛰어지고(skip)**, `getline`에서는 **읽혀서 제거**됩니다.

### getline의 구분문자(delimiter)의 역할: 읽기 종료 기준

```cpp
getline(cin, str, '구분문자');
```

- `getline`은 스트림에서 한 글자씩 읽다가 **구분문자를 만나면 읽기를 멈춥니다.**
- 구분문자는 **버퍼에서 읽어서 제거**되지만, `string str`에는 **저장되지 않습니다.**
- 구분문자를 `'\n'`으로 지정하면 엔터키까지 읽고 멈추는 "한 줄 읽기"가 됩니다.
- 다른 문자(예: `'&'`)로 지정하면 그 문자까지만 읽고, `\n`은 버퍼에 남습니다.

### 비교 예시

```cpp
// 구분문자 = '\n' → 엔터까지 읽고 \n 제거, 버퍼 비워짐
getline(cin, str, '\n');
// 입력: Hello World<Enter>
// str = "Hello World", 버퍼에 아무것도 안 남음

// 구분문자 = '&' → &까지 읽고 & 제거, \n은 버퍼에 남음
getline(cin, str, '&');
// 입력: Hello&<Enter>
// str = "Hello", 버퍼에 '\n' 남음 → 다음 getline('\n')이 빈 문자열을 읽게 됨!
// 이 경우 cin.ignore()로 남은 \n을 제거해야 함
```

### 핵심 차이 요약

| 구분 | 엔터키 | getline 구분문자 |
|------|--------|-----------------|
| 역할 | OS가 입력을 스트림 버퍼에 넣는 신호 | getline이 읽기를 멈추는 기준 |
| 처리 주체 | 운영체제 | getline 함수 |
| 버퍼 처리 | `\n`을 버퍼에 넣음 | 구분문자를 버퍼에서 꺼내고 버림 |
| 구분문자 != `\n`이면 | 버퍼에 `\n`이 남음 | `cin.ignore()` 필요 |
