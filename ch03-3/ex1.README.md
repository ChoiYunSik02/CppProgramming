## 1. 객체의 생성과 소멸이란?

C++에서 **객체(Object)** 란 클래스(Class)를 기반으로 메모리에 실체화된 존재이다.  
객체는 **생성(Creation)** 과 **소멸(Destruction)** 이라는 명확한 생명 주기(Life Cycle)를 가진다.

### 🔷 객체의 생성

객체가 생성되면 다음 두 가지가 순서대로 일어난다.

1. **메모리 할당** : 객체의 데이터 멤버 크기만큼 메모리(스택 or 힙)가 확보된다.
2. **생성자(Constructor) 호출** : 객체의 초기 상태를 설정하는 특수 멤버 함수가 자동 호출된다.

```cpp
class MyClass {
public:
    MyClass() {
        cout << "생성자 호출 → 객체 생성!" << endl;
    }
};

MyClass obj;  // 이 시점에 메모리 할당 + 생성자 자동 호출
```

### 🔷 객체의 소멸

객체가 소멸되면 다음 두 가지가 순서대로 일어난다.

1. **소멸자(Destructor) 호출** : 동적 메모리 해제, 파일 닫기 등 정리 작업을 수행하는 특수 멤버 함수가 자동 호출된다.
2. **메모리 반환** : 할당되었던 메모리가 시스템에 반환된다.

```cpp
class MyClass {
public:
    ~MyClass() {
        cout << "소멸자 호출 → 객체 소멸!" << endl;
    }
};
```

> 💡 **핵심 요약** : 객체의 생성 = 메모리 할당 + 생성자 호출 / 객체의 소멸 = 소멸자 호출 + 메모리 반환

## 2. 지역 변수(지역 객체) vs 전역 변수(전역 객체) 비교

| 비교 항목 | 지역 변수 / 지역 객체 | 전역 변수 / 전역 객체 |
|:---------:|:--------------------:|:--------------------:|
| **선언 위치** | 함수, 블록(`{ }`) 내부 | 모든 함수 외부 (파일 최상단) |
| **접근 범위** | 선언된 블록 내부에서만 접근 가능 | 프로그램 전체 어디서든 접근 가능 |
| **생성 시점** | 선언문이 실행될 때 (런타임) | `main()` 실행 전, 프로그램 시작 시 |
| **소멸 시점** | 선언된 블록(`{ }`)이 종료될 때 | `main()` 종료 후, 프로그램 종료 직전 |
| **자동 초기화** | ❌ 초기화하지 않으면 쓰레기 값 | ✅ 자동으로 0(또는 nullptr)으로 초기화 |
| **저장 영역** | 스택(Stack) 영역 | BSS 또는 데이터(Data) 세그먼트 |
| **생성자 호출** | 선언 시점에 호출 | `main()` 호출 전에 호출 |
| **소멸자 호출** | 블록 종료 시 호출 | `main()` 종료 후 호출 |

### 🔷 예제 코드

```cpp
#include <iostream>
using namespace std;

class Demo {
    string name;
public:
    Demo(string n) : name(n) {
        cout << "[생성] " << name << endl;
    }
    ~Demo() {
        cout << "[소멸] " << name << endl;
    }
};

Demo globalObj("전역 객체");   // 전역 객체 선언

int main() {
    cout << "=== main 시작 ===" << endl;
    Demo localObj("지역 객체"); // 지역 객체 선언
    cout << "=== main 종료 ===" << endl;
    return 0;
    // localObj 소멸 → globalObj 소멸 순서
}
```

### 🔷 실행 결과

```
[생성] 전역 객체       ← main 실행 전에 생성
=== main 시작 ===
[생성] 지역 객체       ← main 내부 선언 시점에 생성
=== main 종료 ===
[소멸] 지역 객체       ← 블록 종료 시 소멸 (LIFO 순서)
[소멸] 전역 객체       ← 프로그램 종료 직전 소멸
```

> 💡 지역 객체는 **생성된 역순(LIFO)** 으로 소멸된다.

## 3. main() 실행 전에 일어나는 일

`main()` 함수가 호출되기 전, C++ 런타임은 **시작 코드(Startup Code / CRT: C Runtime)** 를 실행한다.  
이 과정은 OS가 프로그램을 메모리에 올린 직후부터 시작된다.

### 🔷 처리 순서

```
프로그램 실행
    ↓
① OS → 프로그램 이미지(코드, 데이터) 메모리 적재
    ↓
② CRT 시작 함수 실행 (_start, mainCRTStartup 등)
    ↓
③ BSS 영역 초기화 (전역/정적 변수 → 0으로 초기화)
    ↓
④ 전역 객체 / 정적 객체의 생성자 호출 (선언 순서대로)
    ↓
⑤ 커맨드 라인 인수(argc, argv) 파싱
    ↓
⑥ main() 함수 호출
```

### 🔷 각 단계 설명

| 단계 | 내용 |
|:----:|:-----|
| ① 메모리 적재 | OS가 실행 파일을 읽어 코드, 데이터, BSS, 스택 세그먼트를 메모리에 배치한다. |
| ② CRT 시작 함수 | 컴파일러가 삽입한 `_start` 또는 `mainCRTStartup` 같은 진입점 함수가 실행된다. |
| ③ BSS 초기화 | 초기값이 없는 전역/정적 변수 영역(BSS)을 0으로 클리어한다. |
| ④ 전역 객체 생성자 | 전역/정적 객체의 생성자가 **선언 순서대로** 호출된다. |
| ⑤ 인수 파싱 | `argc`, `argv`, `envp` 등이 설정된다. |
| ⑥ main() 호출 | 드디어 우리가 작성한 `main()` 함수가 실행된다. |

> 💡 **전역 객체 여러 개가 있을 때**: 같은 파일 내에서는 **선언 순서대로** 생성자가 호출된다.  
> 단, 서로 다른 번역 단위(파일) 간의 초기화 순서는 **보장되지 않는다** (Static Initialization Order Fiasco).

## 4. main() 종료 후 프로그램 종료 전에 일어나는 일

`return 0;` 또는 `exit()` 가 호출되면 `main()` 이 종료된다.  
하지만 프로그램이 즉시 끝나는 것이 아니라, 다음과 같은 정리 작업이 수행된다.

### 🔷 처리 순서

```
main() 종료 (return or exit())
    ↓
① 지역 객체 소멸자 호출 (main 내부 객체, LIFO 순서)
    ↓
② atexit() / at_quick_exit()에 등록된 함수 실행 (등록 역순)
    ↓
③ 전역/정적 객체 소멸자 호출 (생성 역순 = LIFO)
    ↓
④ C++ 런타임 정리 (스트림 플러시, 버퍼 비우기 등)
    ↓
⑤ OS에 제어권 반환 (프로세스 종료)
```

### 🔷 각 단계 설명

| 단계 | 내용 |
|:----:|:-----|
| ① 지역 객체 소멸 | `main()` 내 지역 객체들이 생성 역순으로 소멸자 호출된다. |
| ② atexit() 함수 | `atexit(함수포인터)` 로 등록해 둔 함수들이 **등록 역순**으로 실행된다. |
| ③ 전역 객체 소멸 | 전역/정적 객체의 소멸자가 **생성 역순(LIFO)** 으로 호출된다. |
| ④ CRT 정리 | `cout`, `cerr` 등 표준 스트림의 버퍼가 비워지고 플러시된다. |
| ⑤ 프로세스 종료 | OS에 종료 코드(`return`값)를 전달하고 프로세스가 완전히 종료된다. |

### 🔷 atexit() 예제

```cpp
#include <iostream>
#include <cstdlib>
using namespace std;

void cleanup1() { cout << "cleanup1 실행" << endl; }
void cleanup2() { cout << "cleanup2 실행" << endl; }

int main() {
    atexit(cleanup1);  // 먼저 등록
    atexit(cleanup2);  // 나중에 등록
    cout << "main 종료" << endl;
    return 0;
}
```

```
main 종료
cleanup2 실행   ← 나중에 등록한 것이 먼저 실행 (역순)
cleanup1 실행
```

## 5. 전체 프로그램 생명 주기 요약

```
┌─────────────────────────────────────────────────────────┐
│                   프로그램 실행 시작                      │
│  ① OS 메모리 적재                                        │
│  ② CRT 시작 코드 실행                                    │
│  ③ BSS 영역 0 초기화                                     │
│  ④ 전역/정적 객체 생성자 호출 (선언 순서)                │
│  ⑤ main() 호출                                          │
│ ┌───────────────────────────────────────────────────┐   │
│ │  main() 실행 중                                    │   │
│ │   - 지역 객체 생성 (선언 시점)                     │   │
│ │   - 지역 객체 소멸 (블록 종료, LIFO)               │   │
│ └───────────────────────────────────────────────────┘   │
│  ⑥ main() 종료                                          │
│  ⑦ atexit() 등록 함수 실행 (역순)                       │
│  ⑧ 전역/정적 객체 소멸자 호출 (생성 역순)              │
│  ⑨ CRT 정리 (스트림 플러시 등)                          │
│  ⑩ OS에 종료 코드 반환                                  │
└─────────────────────────────────────────────────────────┘
```

---

> 📌 **참고**: `exit()` 함수는 `atexit()` 및 전역 객체 소멸자를 호출하지만,  
> `_exit()` 또는 `abort()` 는 이 정리 과정을 **건너뛰고** 즉시 종료한다.
