# 📘 객체의 생성과 소멸, 지역/전역 변수 비교, main 전후 처리

---

## 1. 객체의 생성과 소멸이란?

C++에서 **객체(Object)** 란 클래스(Class)를 기반으로 메모리에 실체화된 존재이다. 객체는 **생성(Creation)** 과 **소멸(Destruction)** 이라는 명확한 생명 주기(Life Cycle)를 가진다.

### 🔷 객체의 생성

객체가 생성된다는 것은 두 가지 과정이 순서대로 이루어지는 것을 의미한다.

첫째로 **메모리 할당**이 일어난다. 객체의 데이터 멤버들이 저장될 수 있도록 그 크기만큼의 메모리 공간이 스택 또는 힙 영역에 확보된다.

둘째로 **생성자(Constructor)가 자동으로 호출**된다. 생성자는 객체의 초기 상태를 설정하기 위해 프로그래머가 정의한 특수 멤버 함수로, 객체가 선언되는 순간 컴파일러가 자동으로 호출한다. 생성자를 따로 정의하지 않았다면 컴파일러가 기본 생성자를 자동으로 제공한다.

### 🔷 객체의 소멸

객체가 소멸된다는 것 역시 두 가지 과정으로 이루어진다.

첫째로 **소멸자(Destructor)가 자동으로 호출**된다. 소멸자는 객체가 사용하던 동적 메모리를 해제하거나, 열었던 파일을 닫는 등 뒷정리 작업을 수행하는 특수 멤버 함수이다. 객체의 수명이 끝나는 시점에 컴파일러가 자동으로 호출한다.

둘째로 **메모리가 반환**된다. 소멸자 호출이 완료되면 객체가 사용하던 메모리 공간이 시스템에 반환된다.

> 💡 **핵심 요약** : 객체의 생성 = 메모리 할당 + 생성자 호출 / 객체의 소멸 = 소멸자 호출 + 메모리 반환

## 2. 지역 변수(지역 객체) vs 전역 변수(전역 객체) 비교

| 비교 항목 | 지역 변수 / 지역 객체 | 전역 변수 / 전역 객체 |
|:---------:|:--------------------:|:--------------------:|
| **선언 위치** | 함수, 블록(`{ }`) 내부 | 모든 함수 외부 (파일 최상단) |
| **접근 범위** | 선언된 블록 내부에서만 접근 가능 | 프로그램 전체 어디서든 접근 가능 |
| **생성 시점** | 선언문이 실행되는 시점 (런타임) | `main()` 실행 전, 프로그램 시작 시 |
| **소멸 시점** | 선언된 블록(`{ }`)이 종료될 때 | `main()` 종료 후, 프로그램 종료 직전 |
| **자동 초기화** | ❌ 초기화하지 않으면 쓰레기 값 | ✅ 자동으로 0(또는 nullptr)으로 초기화 |
| **저장 영역** | 스택(Stack) 영역 | BSS 또는 데이터(Data) 세그먼트 |
| **생성자 호출** | 선언 시점에 호출 | `main()` 호출 전에 호출 |
| **소멸자 호출** | 블록 종료 시 호출 | `main()` 종료 후 호출 |

### 🔷 선언 위치

지역 변수는 함수나 블록(`{ }`) 내부에 선언된다. 반면 전역 변수는 모든 함수의 바깥, 즉 파일의 최상단에 선언된다.

### 🔷 접근 범위

지역 변수는 자신이 선언된 블록 안에서만 접근할 수 있다. 블록 밖에서 해당 변수에 접근하려 하면 컴파일 오류가 발생한다. 전역 변수는 프로그램 전체 어느 함수에서든 이름만으로 접근할 수 있다.

### 🔷 생성 시점

지역 객체는 프로그램 실행 중 해당 선언문이 실제로 실행되는 시점에 생성된다. 전역 객체는 `main()` 함수가 시작되기도 전에, 프로그램이 시작하는 시점에 이미 생성된다.

### 🔷 소멸 시점

지역 객체는 자신이 속한 블록(`{ }`)이 끝나는 순간 소멸된다. 여러 지역 객체가 있다면 생성된 순서의 역순(LIFO: Last In, First Out)으로 소멸된다. 전역 객체는 `main()` 함수가 종료된 이후, 프로그램이 완전히 끝나기 직전에 소멸된다.

### 🔷 초기화 여부

지역 변수는 선언 시 초기값을 명시하지 않으면 메모리에 남아있던 쓰레기 값(garbage value)이 그대로 들어있다. 반드시 직접 초기화해야 한다. 반면 전역 변수는 초기값을 지정하지 않아도 컴파일러가 자동으로 0 또는 nullptr로 초기화해 준다.

## 3. main() 실행 전에 일어나는 일

`main()` 함수가 호출되기 전, C++ 런타임은 **시작 코드(Startup Code / CRT: C Runtime)** 를 실행한다. 사용자 눈에는 보이지 않지만, 프로그램이 실행 가능한 상태가 되기까지 다음과 같은 과정이 순서대로 처리된다.

### ① OS의 프로그램 메모리 적재

운영체제(OS)가 실행 파일을 읽어 코드 영역, 데이터 영역, BSS 영역, 스택 영역 등을 메모리에 배치한다. 이 단계에서 프로그램이 실행될 기본 환경이 갖춰진다.

### ② CRT 시작 함수 실행

컴파일러가 실행 파일에 자동으로 삽입해 놓은 `_start` 또는 `mainCRTStartup` 같은 진입점 함수가 실행된다. 이 함수가 `main()`을 대신해 가장 먼저 호출되는 진짜 시작점이며, 이후 모든 준비 작업을 담당한다.

### ③ BSS 영역 초기화

초기값이 지정되지 않은 전역 변수와 정적(static) 변수가 저장되는 BSS 영역 전체를 0으로 초기화한다. 이 덕분에 전역 변수는 명시적으로 초기화하지 않아도 자동으로 0이 된다.

### ④ 전역 객체와 정적 객체의 생성자 호출

전역 객체 및 정적 객체의 생성자가 **선언된 순서대로** 호출된다. 같은 파일 안에서는 선언 순서가 보장되지만, 서로 다른 파일에 선언된 전역 객체 간의 초기화 순서는 표준에서 보장하지 않는다. 이를 **정적 초기화 순서 문제(Static Initialization Order Fiasco)** 라고 부른다.

### ⑤ 커맨드 라인 인수 파싱

프로그램 실행 시 전달된 커맨드 라인 인수들이 파싱되어 `argc`와 `argv` 변수에 설정된다.

### ⑥ main() 호출

위의 모든 준비가 완료된 후, 비로소 우리가 작성한 `main()` 함수가 호출된다.

## 4. main() 종료 후 프로그램 종료 전에 일어나는 일

`return` 문 실행 또는 `exit()` 함수 호출로 `main()`이 종료되더라도 프로그램이 즉시 끝나는 것은 아니다. 다음과 같은 정리(cleanup) 작업이 자동으로 순서대로 수행된다.

### ① 지역 객체 소멸자 호출

`main()` 내부에 선언된 지역 객체들의 소멸자가 생성된 역순(LIFO)으로 호출된다.

### ② atexit()에 등록된 함수 실행

`atexit()` 함수를 통해 사전에 등록해 둔 콜백 함수들이 **등록된 역순**으로 실행된다. 프로그래머가 프로그램 종료 시 반드시 실행되어야 하는 정리 코드를 등록해 두는 용도로 활용한다.

### ③ 전역/정적 객체 소멸자 호출

전역 객체와 정적 객체의 소멸자가 생성된 순서의 역순(LIFO)으로 호출된다. 즉, 가장 마지막에 생성된 전역 객체가 가장 먼저 소멸된다.

### ④ C++ 런타임 정리

`cout`, `cerr` 등 표준 스트림의 내부 버퍼가 모두 비워지고 플러시(flush)된다. 스트림이 정상적으로 닫히고 런타임 관련 자원들이 정리된다.

### ⑤ OS에 제어권 반환

`main()`이 반환한 값(종료 코드)을 운영체제에 전달하고, 프로세스가 완전히 종료된다.

---

## 5. 전체 프로그램 생명 주기 요약

```
프로그램 실행 시작
    ↓
① OS → 프로그램 메모리 적재
    ↓
② CRT 시작 함수 실행 (_start 등)
    ↓
③ BSS 영역 0 초기화 (전역/정적 변수)
    ↓
④ 전역/정적 객체 생성자 호출 (선언 순서)
    ↓
⑤ main() 호출
    ↓
[ main() 실행 중 ]
  - 지역 객체 생성 (선언 시점)
  - 지역 객체 소멸 (블록 종료, LIFO)
    ↓
⑥ main() 종료
    ↓
⑦ atexit() 등록 함수 실행 (등록 역순)
    ↓
⑧ 전역/정적 객체 소멸자 호출 (생성 역순)
    ↓
⑨ CRT 정리 (스트림 플러시 등)
    ↓
⑩ OS에 종료 코드 반환 → 프로세스 종료
```

---

> 📌 **참고** : `exit()` 함수는 `atexit()` 및 전역 객체 소멸자를 포함한 정리 과정을 정상적으로 수행하지만, `_exit()` 또는 `abort()` 는 이 모든 정리 과정을 **건너뛰고** 즉시 종료한다.
