# 📘 객체의 생성과 소멸, 지역/전역 변수 비교, main 전후 처리

---

## 1. 객체의 생성과 소멸의 의미

### 🔷 객체 생성의 의미

객체를 생성하거나 변수를 선언하면 **메모리를 할당** 받는다. 이는 운영체제가 관리하는 메모리 공간을 프로그램에서 사용할 수 있게 됨을 의미한다. 동시에 생성자(Constructor)가 자동으로 호출되어 객체의 초기 상태가 설정된다.

### 🔷 객체 소멸의 의미

함수가 종료되거나 프로그램이 종료되면 객체가 소멸된다. 이는 프로그램에서 사용하던 메모리 공간을 **운영체제에 반납**하고 더 이상 그 공간을 사용할 수 없게 됨을 의미한다. 객체가 소멸되기 직전에는 소멸자(Destructor)가 자동으로 호출되어 뒷정리 작업이 수행된다.

### 🔷 생성자와 소멸자

**생성자**는 객체가 생성될 때 자동으로 한 번 호출되는 멤버 함수이다. 클래스와 이름이 동일하며, 멤버 변수의 초기화나 필요한 준비 작업을 담당한다.

**소멸자**는 객체가 메모리에서 소멸되기 직전에 자동으로 한 번 호출되는 멤버 함수이다. 클래스 이름 앞에 `~`(틸드)를 붙여 표기하며, 리턴 타입과 매개변수가 없다. 클래스 내에 오직 하나만 존재할 수 있고, 사용자가 임의로 호출할 수 없다. 동적으로 할당받은 메모리 해제, 파일 저장 및 닫기 등의 마무리 작업을 처리한다.

만약 개발자가 소멸자를 따로 작성하지 않으면, 컴파일러가 매개변수가 없고 아무 일도 하지 않는 **디폴트 소멸자**를 자동으로 추가한다.

## 2. 지역 변수(지역 객체) vs 전역 변수(전역 객체) 비교

| 비교 항목 | 지역 변수 / 지역 객체 | 전역 변수 / 전역 객체 |
|:---------:|:--------------------:|:--------------------:|
| **선언 위치** | 함수 내부 | 함수 외부 |
| **접근 범위** | 선언된 함수(블록) 내부에서만 접근 가능 | 프로그램 전체 어디서든 접근 가능 |
| **생성 시점** | 함수가 호출될 때 선언된 순서대로 생성 | 프로그램 실행 시 main 함수 실행 전에 선언된 순서대로 생성 |
| **소멸 시점** | 함수가 종료될 때 생성된 순서의 역순으로 소멸 | 프로그램(main 함수)이 종료될 때 생성된 순서의 역순으로 소멸 |
| **자동 초기화** | ❌ 초기화하지 않으면 쓰레기 값 | ✅ 자동으로 0으로 초기화 |
| **저장 영역** | 스택(Stack) 영역 | 데이터(Data) 세그먼트 |

### 🔷 선언 위치

지역 객체는 함수 내부에 선언된 객체로서 지역변수와 같다. 전역 객체는 함수의 외부에 선언된 객체로서 전역변수와 같다.

### 🔷 접근 범위

지역 변수는 자신이 선언된 함수(블록) 안에서만 접근할 수 있다. 전역 변수는 프로그램 전체 어느 함수에서든 접근할 수 있다.

### 🔷 생성 시점

전역 객체는 프로그램이 실행될 때 main 함수 실행 전에 선언된 순서대로 생성된다. 지역 객체는 해당 함수가 호출될 때 선언된 순서대로 생성된다.

### 🔷 소멸 시점

지역 객체는 함수가 종료되면 생성된 순서의 역순으로 소멸된다. 지역 객체는 스택(Stack) 메모리에 할당되기 때문에 LIFO(Last In, First Out) 구조에 따라 마지막에 생성된 것이 가장 먼저 소멸된다. 전역 객체는 프로그램(main 함수)이 종료될 때 생성된 순서의 역순으로 소멸된다.

### 🔷 초기화 여부

지역 변수는 선언 시 초기값을 명시하지 않으면 쓰레기 값(garbage value)이 그대로 남아있다. 전역 변수는 초기값을 지정하지 않아도 자동으로 0으로 초기화된다.

## 3. main 함수 실행 전에 일어나는 일

프로그램이 실행되면 main 함수가 곧바로 시작되는 것이 아니다. 그 이전에 다음과 같은 초기화 작업이 순서대로 처리된다.

**① 운영체제가 프로그램을 메모리에 적재한다.**
실행 파일을 읽어 코드, 데이터, 스택 등 각 영역을 메모리에 배치하여 실행 환경을 구성한다.

**② 전역 변수와 정적 변수가 0으로 초기화된다.**
초기값이 지정되지 않은 전역 변수와 정적(static) 변수들이 자동으로 0으로 초기화된다. 이것이 전역 변수가 명시적으로 초기화하지 않아도 0이 되는 이유이다.

**③ 전역 객체의 생성자가 호출된다.**
소스 코드에서 전역 객체가 선언된 순서대로 생성자가 호출된다. 이 과정이 바로 강의 자료의 다이어그램에서 "초기화 작업 → 전역변수, 전역객체 생성"에 해당하는 단계이다.

**④ main 함수가 호출된다.**
위의 모든 준비가 완료된 후, 비로소 우리가 작성한 main 함수가 실행된다.

---

## 4. main 함수 종료 후 프로그램 종료 전에 일어나는 일

main 함수의 return 문이 실행되어 main이 종료되더라도 프로그램이 즉시 끝나지 않는다. 다음과 같은 정리(cleanup) 작업이 자동으로 수행된다.

**① main 함수 내 지역 객체의 소멸자가 호출된다.**
main 함수 안에 선언된 지역 객체들의 소멸자가 생성된 역순으로 호출된다.

**② 전역 객체의 소멸자가 호출된다.**
프로그램 전체에 선언된 전역 객체들의 소멸자가 생성된 역순으로 호출된다. 이 과정이 강의 자료 다이어그램에서 "정리 작업 → 전역변수, 전역객체 소멸"에 해당하는 단계이다.

**③ 프로그램이 완전히 종료된다.**
모든 정리 작업이 끝나면 운영체제에 제어권이 반환되고 프로세스가 완전히 종료된다.

## 5. 전체 실행 흐름 요약

```
프로그램 실행 시작
    ↓
① 전역변수, 전역객체 생성 (선언 순서대로)     ← main 함수 실행 전
    ↓
② main 함수 시작 → 지역변수, 지역객체 생성
    ↓
③ 함수1 호출 → 지역변수, 지역객체 생성
    ↓
④ 함수1 종료 → 지역변수, 지역객체 소멸 (역순)
    ↓
  ... (다른 함수들도 동일하게 반복)
    ↓
⑤ main 함수 종료 → 지역변수, 지역객체 소멸 (역순)
    ↓
⑥ 전역변수, 전역객체 소멸 (생성 역순)         ← 프로그램 종료 직전
    ↓
프로그램 종료
```
