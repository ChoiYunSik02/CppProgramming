## 소스코드 설명

`#include <iostream>` : 표준 입출력 기능을 사용하기 위해 iostream 헤더 파일을 포함한다.

`using namespace std;` : std 네임스페이스를 사용 선언하여 `cout`, `endl` 앞에 `std::` 를 붙이지 않아도 되도록 한다.

`class Triangle` : Triangle 이라는 이름의 클래스를 정의한다.

`public:` : 이하 멤버들을 외부에서 자유롭게 접근할 수 있도록 public 접근 지정자를 선언한다.

`int width;` : 삼각형의 밑변 값을 저장하는 정수형 멤버 변수를 선언한다.

`int height;` : 삼각형의 높이 값을 저장하는 정수형 멤버 변수를 선언한다.

`Triangle();` : 매개변수 없이 기본값으로 객체를 생성하는 생성자1을 선언한다.

`Triangle(int a, int b);` : 밑변과 높이를 매개변수로 받아 객체를 생성하는 생성자2를 선언한다.

`~Triangle();` : 객체가 소멸될 때 자동으로 호출되는 소멸자를 선언한다.

`double getArea();` : 삼각형의 면적을 계산하여 실수형으로 반환하는 멤버 함수를 선언한다.

`Triangle::Triangle() : Triangle(1, 1) { }` : 매개변수 없는 기본 생성자(위임 생성자)를 구현한다. 멤버 초기화 리스트에서 `Triangle(1, 1)` 을 호출하여 초기화 작업 전체를 타겟 생성자에게 위임한다. 별도로 직접 초기화하거나 출력하는 코드 없이 중괄호 내부가 비어 있다.

`Triangle::Triangle(int a, int b) : width(a), height(b)` : 밑변과 높이를 매개변수로 받는 타겟 생성자를 구현한다. 멤버 초기화 리스트 `: width(a), height(b)` 를 사용하여 매개변수로 전달받은 값을 멤버 변수에 직접 초기화한다.

`cout << "밑변 " << width << " 높이 " << height << " 인 삼각형 생성" << endl;` : 객체가 생성될 때 밑변과 높이를 화면에 출력하여 생성 사실을 알린다. 위임 생성자가 호출될 때도 최종적으로 이 타겟 생성자가 실행되므로 출력은 항상 이 줄에서만 발생한다.

`Triangle::~Triangle()` : 소멸자를 구현한다. 객체가 메모리에서 소멸되기 직전에 자동으로 한 번 호출된다.

`cout << "밑변 " << width << " 높이 " << height << " 인 삼각형 소멸" << endl;` : 소멸자 내부에서 해당 객체의 밑변과 높이를 출력하여 소멸 사실을 알린다.

`double Triangle::getArea()` : 삼각형 면적을 계산하여 반환하는 함수를 구현한다.

`return (1.0 / 2.0) * width * height;` : 삼각형 넓이 공식인 `(밑변 × 높이) ÷ 2` 를 계산하여 반환한다. 정수 나눗셈으로 인한 소수점 손실을 막기 위해 `1 / 2` 대신 `1.0 / 2.0` 으로 작성하여 실수 연산이 되도록 한다.

`Triangle tri1;` : 매개변수 없는 위임 생성자를 호출하여 tri1 객체를 생성한다. 위임 생성자가 내부적으로 `Triangle(1, 1)` 을 호출하므로 밑변과 높이가 모두 1로 초기화된다.

`cout << "삼각형의 면적은 " << tri1.getArea() << endl;` : tri1 객체의 getArea() 함수를 호출하여 면적을 계산하고 결과를 출력한다. 면적은 `(1 × 1) ÷ 2 = 0.5` 이다.

`Triangle tri2(2, 4);` : 밑변 2, 높이 4를 인수로 넘겨 타겟 생성자를 직접 호출하여 tri2 객체를 생성한다. 멤버 초기화 리스트에 의해 width = 2, height = 4 로 초기화된다.

`cout << "삼각형의 면적은 " << tri2.getArea() << endl;` : tri2 객체의 getArea() 함수를 호출하여 면적을 계산하고 결과를 출력한다. 면적은 `(2 × 4) ÷ 2 = 4` 이다.

`return 0;` : main 함수가 종료된다. 지역 객체들은 생성된 역순(LIFO)으로 소멸되므로 tri2의 소멸자가 먼저 호출된 후 tri1의 소멸자가 호출된다.
