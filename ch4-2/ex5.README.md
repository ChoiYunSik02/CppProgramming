# ch4-2장 실습과제 5번 - 소스코드 설명

---

## 1. 과제 설명

저장할 문자열의 크기를 먼저 입력 받아 **동적 할당(new)** 을 이용하여 메모리를 할당하고,  
공백이 포함된 문자열을 입력 받아 출력하는 프로그램이다.  
동적 할당 시 **널문자(`'\0'`)를 저장할 공간까지 고려**하여 `n+1` 크기로 할당해야 한다.

---

## 2. 코드 흐름 (단계별 설명)

### Step 1. 문자열 크기 입력 및 유효성 검사

```cpp
int n;
cin >> n;
if (n <= 0) return -1;
```

- 저장할 문자열의 길이 `n`을 입력 받는다.
- `n`이 0 이하이면 의미 없는 입력이므로 즉시 종료한다.
- 여기서 `n`은 **순수 문자 개수**이며, 실제 할당 크기는 `n+1`이 된다.

---

### Step 2. 동적 메모리 할당 (+1의 의미)

```cpp
char* p = new char[n + 1];
```

- `char` 배열로 문자열을 저장할 때는 마지막에 **널문자(`'\0'`)** 가 반드시 필요하다.
- 널문자는 문자열의 끝을 알리는 신호로, 이 공간이 없으면 문자열이 어디서 끝나는지 알 수 없다.
- 따라서 문자 `n`개를 저장하려면 `n+1` 크기로 할당해야 한다.

#### "I am a boy." 를 예시로 한 메모리 구조

```
입력한 n = 11

힙(Heap) - new char[11+1] = 12칸 할당
┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬────┐
│ I │   │ a │ m │   │ a │   │ b │ o │ y │ . │ \0 │
└───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴────┘
 p[0]                                        p[11]
 ←────────── 문자 11개 ──────────────────→  ← 널문자
```

> ⚠️ `new char[n]`으로 할당하면 널문자 공간이 부족해 **메모리 오염**이 발생할 수 있다.

---

### Step 3. cin.ignore() - 입력 버퍼 정리

```cpp
cin.ignore();
```

- `cin >> n`으로 숫자를 입력하면, 사용자가 누른 **엔터(`\n`)가 입력 버퍼에 그대로 남는다.**
- `cin.ignore()`를 호출하지 않으면, 이후 `getline`이 버퍼에 남은 `\n`을 **문자열 입력으로 착각**하고 빈 문자열을 저장한 뒤 바로 종료해버린다.

#### cin.ignore() 없을 때 vs 있을 때

```
【 cin.ignore() 없을 때 】
사용자 입력: 11 + 엔터
버퍼 상태: [ 1 ][ 1 ][ \n ]
cin >> n 실행: "11" 가져감 → 버퍼: [ \n ] 남음
getline 실행: "\n 있네? 입력 끝!" → 빈 문자열 저장 ❌

【 cin.ignore() 있을 때 】
사용자 입력: 11 + 엔터
버퍼 상태: [ 1 ][ 1 ][ \n ]
cin >> n 실행: "11" 가져감 → 버퍼: [ \n ] 남음
cin.ignore() 실행: [ \n ] 제거 → 버퍼: 비워짐
getline 실행: "버퍼 비었으니 사용자 입력 대기"
사용자 입력: "I am a boy." → 정상 저장 ✅
```

---

### Step 4. cin.getline() - 공백 포함 문자열 입력

```cpp
cin.getline(p, n + 1);
```

- `cin >>` 는 **공백을 기준으로 입력을 끊기** 때문에 `"I am a boy."`처럼 공백이 포함된 문자열을 읽을 수 없다.
- `cin.getline(포인터, 최대크기)`는 **엔터 전까지 공백을 포함한 전체 문자열**을 읽는다.
- 두 번째 인자 `n+1`은 읽을 수 있는 최대 문자 수(널문자 포함)를 지정한다.

#### cin >> 와 cin.getline() 비교

| 구분 | `cin >> p` | `cin.getline(p, n+1)` |
|:----:|:----------:|:---------------------:|
| **공백 처리** | 공백에서 입력 중단 | 공백 포함 전체 입력 |
| **입력 예시** | `"I"` 만 저장 | `"I am a boy."` 저장 |
| **널문자** | 자동 추가 | 자동 추가 |
| **최대 크기 지정** | 불가 | 가능 (버퍼 오버플로 방지) |

---

### Step 5. 동적 메모리 반환

```cpp
delete[] p;
```

- `new[]`로 할당한 배열은 반드시 **`delete[]`** 로 반환해야 한다.
- 반환하지 않으면 프로그램이 종료될 때까지 힙 메모리가 점유되는 **메모리 누수(Memory Leak)** 가 발생한다.

---

## 3. 핵심 개념 요약

| 개념 | 설명 |
|:----:|:-----|
| `new char[n+1]` | 문자 n개 + 널문자 1개 공간을 힙에 동적 할당 |
| `'\0'` (널문자) | 문자열의 끝을 표시, 값은 0, 반드시 마지막 칸에 저장됨 |
| `cin.ignore()` | 입력 버퍼에 남은 엔터(`\n`) 제거 |
| `cin.getline(p, n+1)` | 공백 포함 문자열을 최대 n+1글자까지 읽음 |
| `delete[] p` | 동적 할당 배열 메모리 반환 (`[]` 필수) |
