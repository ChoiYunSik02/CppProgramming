# 실습과제2 - NULL 매크로상수와 nullptr 정리

---

## 1. NULL이란?

**NULL**은 포인터가 **아무것도 가리키지 않음**을 나타내기 위해 사용하는 매크로 상수이다.  
즉, 유효하지 않은 메모리 주소를 나타낼 때 사용하며, 포인터 변수를 초기화하거나 포인터가 유효한지 검사할 때 활용한다.

```cpp
int* p = new int;
if (p == NULL) {
    cout << "메모리 할당 실패" << endl;
}
```

---

## 2. C와 C++에서 NULL 정의의 차이

| 구분 | 정의 방식 | 설명 |
|:----:|:---------:|:-----|
| **C 언어** | `#define NULL ((void*)0)` | 정수 0을 void 포인터로 캐스팅한 값 |
| **C++ 언어** | `#define NULL 0` | 단순히 정수 0으로 정의 |

### C에서의 NULL
```c
// C 언어 (stddef.h 기준)
#define NULL ((void*)0)

int* p = NULL; // void* 타입의 0, 즉 null 포인터
```

### C++에서의 NULL
```cpp
// C++ 언어 (iostream 등 기준)
#define NULL 0

int* p = NULL; // 정수 0으로 정의됨
```

> C++에서 NULL을 `((void*)0)`으로 정의하지 않는 이유:  
> C++은 `void*`를 다른 포인터 타입으로 **암묵적 변환을 허용하지 않기** 때문이다.

---

## 3. NULL이 정의된 헤더파일

NULL은 여러 헤더파일에서 정의되어 있으며, 아래 중 하나를 포함하면 사용할 수 있다.

| 헤더파일 | 비고 |
|:--------:|:-----|
| `<stddef.h>` (C) / `<cstddef>` (C++) | 가장 기본적인 정의 헤더 |
| `<stdio.h>` (C) / `<cstdio>` (C++) | C 표준 입출력 헤더 |
| `<stdlib.h>` (C) / `<cstdlib>` (C++) | C 표준 유틸리티 헤더 |
| `<string.h>` (C) / `<cstring>` (C++) | 문자열 처리 헤더 |
| `<iostream>` (C++) | C++ 입출력 헤더 (간접 포함) |

```cpp
#include <iostream>  // NULL 사용 가능
// 또는
#include <cstddef>   // NULL 명시적 포함
```

---

## 4. 널문자(`'\0'`)와 NULL의 차이

널문자와 NULL은 **값은 같지만(0) 의미와 사용 목적이 전혀 다르다.**

| 구분 | 널문자 (`'\0'`) | NULL |
|:----:|:---------------:|:----:|
| **타입** | `char` 타입 (1바이트) | 포인터 타입 (주소값) |
| **값** | 0 | 0 |
| **용도** | 문자열의 끝을 표시 | 포인터가 아무것도 가리키지 않음을 표시 |
| **사용 위치** | 문자 배열(문자열)의 마지막 | 포인터 변수 |

```cpp
// 널문자('\0') - 문자열 종료 표시
char str[6] = "Hello";
// 메모리: 'H' 'e' 'l' 'l' 'o' '\0'
//                                 ↑ 문자열의 끝

// NULL - 포인터가 아무것도 가리키지 않음
int* p = NULL;  // 포인터 초기화
if (p == NULL) {
    cout << "유효하지 않은 포인터" << endl;
}
```

> ⚠️ 혼동 주의: `'\0'`은 **문자열 처리**에, `NULL`은 **포인터 처리**에 사용한다.

---

## 5. NULL의 사용 용도

### ① 포인터 초기화
```cpp
int* p = NULL;  // 포인터를 명시적으로 초기화
```

### ② 동적 할당 실패 검사
```cpp
int* p = new int;
if (p == NULL) {        // 또는 if (!p)
    cout << "메모리 부족" << endl;
    return -1;
}
```

### ③ 함수 반환값 검사
```cpp
char* result = strchr(str, 'a');
if (result == NULL) {
    cout << "문자를 찾지 못했습니다." << endl;
}
```

## 6. nullptr (C++11 이후 권장)

### nullptr이란?
C++11 표준부터 도입된 **포인터 전용 리터럴(literal)**로,  
기존 `NULL`(정수 0)의 문제점을 해결하기 위해 만들어졌다.

- 타입: `std::nullptr_t` (포인터 전용 타입)
- 별도의 헤더파일 없이 사용 가능 (언어 자체 키워드)

```cpp
int* p = nullptr;  // C++11 이후 권장 방식
```

---

### NULL의 문제점 (nullptr이 필요한 이유)

NULL은 내부적으로 정수 `0`이므로, **함수 오버로딩** 시 의도치 않은 함수가 호출되는 문제가 발생한다.

```cpp
void func(int n) {
    cout << "정수형 함수 호출" << endl;
}
void func(int* p) {
    cout << "포인터형 함수 호출" << endl;
}

func(NULL);    // ❌ "정수형 함수 호출" -> 의도와 다른 결과!
func(nullptr); // ✅ "포인터형 함수 호출" -> 의도한 결과
```

---

### NULL vs nullptr 비교

| 구분 | `NULL` | `nullptr` |
|:----:|:------:|:---------:|
| **도입 시기** | C 언어부터 | C++11 이후 |
| **타입** | 정수(`int`) | `std::nullptr_t` (포인터 전용) |
| **헤더파일** | `<cstddef>` 등 필요 | 불필요 (언어 내장) |
| **함수 오버로딩** | 정수로 해석되어 오작동 가능 | 포인터로 정확히 해석 |
| **타입 안전성** | 낮음 | 높음 |
| **권장 여부** | C++ 에서 비권장 | ✅ C++11 이후 권장 |

---

### nullptr 사용 예시

```cpp
#include <iostream>
using namespace std;

int main() {
    int* p = nullptr;  // 포인터 초기화

    p = new int;
    if (p == nullptr) {  // 또는 if (!p)
        cout << "메모리 부족" << endl;
        return -1;
    }

    *p = 42;
    cout << *p << endl;

    delete p;
    p = nullptr;  // 해제 후 nullptr로 초기화 (안전한 습관)

    return 0;
}
```

> 💡 **권장 사항**: C++11 이후에는 `NULL` 대신 **`nullptr`을 사용**하는 것이 타입 안전성 측면에서 더 좋다.

---

## 7. 정리

```
NULL      → 정수 0으로 정의된 매크로 상수, 포인터의 무효값 표시 (C 스타일)
'\0'      → 문자열의 끝을 나타내는 널문자 (char 타입, 값 0)
nullptr   → C++11 도입, 포인터 전용 리터럴, 타입 안전성 보장 (C++ 권장)
```
