# 3-4장 실습과제1 - 접근 지정자, 생성자/setter, 캡슐화와 정보은닉

---

## 1. 접근 지정자가 왜 필요한가?

접근 지정자(Access Specifier)는 **객체의 캡슐화를 구현하기 위한 도구**이다.

클래스의 멤버(변수 및 함수)에 대해 **클래스 외부에서의 접근 허용 여부를 제어**한다.

### 필요한 이유

- 멤버 변수를 외부에서 직접 수정할 수 있다면, 잘못된 값이 저장되거나 객체의 상태가 의도치 않게 변경될 수 있다.
- 접근 지정자를 통해 **중요한 멤버는 비공개(private)** 로 보호하고, **외부 인터페이스만 공개(public)** 함으로써 데이터의 무결성을 유지할 수 있다.
- 잘못된 접근이 발생하더라도 **컴파일 과정에서 오류로 검출**되므로 버그를 사전에 방지할 수 있다.

### 접근 지정자 종류 요약

| 접근 지정자 | 접근 범위 |
|---|---|
| `private` | 클래스 내부 멤버 함수에서만 접근 가능 |
| `public` | 클래스 내·외부 어디서든 접근 가능 |
| `protected` | 클래스 자신과 상속받은 자식 클래스에서만 접근 가능 |

## 2. 생성자와 setter 멤버 함수의 차이점

생성자와 setter는 모두 멤버 변수의 값을 설정한다는 공통점이 있지만, **사용 시점과 목적이 다르다.**

### 생성자 (Constructor)

- **객체가 생성되는 순간(최초 1회)** 자동으로 호출된다.
- 멤버 변수의 **초기값을 설정**하는 것이 주된 목적이다.
- 객체가 만들어질 때 반드시 유효한 상태를 갖도록 보장한다.
- 직접 호출할 수 없으며, 객체 생성 시 자동으로 실행된다.

```cpp
Circle::Circle(int r) {
    radius = r;  // 객체 생성 시 초기화
}
```

### setter 멤버 함수

- **객체 생성 이후** 언제든지 멤버 변수의 값을 **변경**하기 위해 호출된다.
- 잘못된 값이 입력되었을 때 **에러 처리 로직**을 포함할 수 있다.
- 명시적으로 호출해야 한다.

```cpp
void Circle::setRadius(int r) {
    if (r < 0)   // 유효성 검사
        return;  // 잘못된 값 거부
    radius = r;
}
```

### 비교 요약

| 구분 | 생성자 | setter |
|---|---|---|
| 호출 시점 | 객체 생성 시 자동 호출 | 객체 생성 이후 명시적 호출 |
| 목적 | 멤버 변수 **초기화** | 멤버 변수 **값 변경** |
| 호출 횟수 | 단 1회 | 필요할 때마다 여러 번 |
| 유효성 검사 | 가능하지만 주 목적은 아님 | 주로 유효성 검사와 함께 사용 |

## 3. 캡슐화와 정보은닉

### 캡슐화 (Encapsulation)

캡슐화란 **데이터(멤버 변수)와 함수(멤버 함수)를 하나의 클래스 안에 묶고, 외부로부터 안전하게 보호**하는 개념이다.

- 클래스는 하나의 캡슐처럼 동작하여 내부 구현을 숨기고, 외부에는 필요한 인터페이스(public 멤버 함수)만 노출한다.
- 덕분에 클래스 내부 구현이 변경되어도 외부 코드에는 영향을 주지 않는다.

```
┌──────────────────────────────┐
│         객체 (캡슐)           │
│  ┌────────────────────────┐  │
│  │  private 멤버 변수      │  │← 외부 접근 불가
│  │  (radius, width, ...)  │  │
│  └────────────────────────┘  │
│  ┌────────────────────────┐  │
│  │  public 멤버 함수       │  │← 외부에서 접근 가능
│  │  (getArea, setRadius)  │  │
│  └────────────────────────┘  │
└──────────────────────────────┘
```

### 정보 은닉 (Information Hiding)

정보 은닉이란 **멤버 변수를 private으로 선언**하고, 해당 변수에 접근하기 위한 **getter/setter 함수를 별도로 제공**하여 안전하게 접근하도록 하는 개념이다.

- 외부에서 멤버 변수에 직접 접근하지 못하게 막음으로써 **잘못된 값의 저장과 사용을 방지**한다.
- setter 함수 내부에서 유효성 검사를 수행하므로 **데이터 오염을 사전에 차단**할 수 있다.
- 컴파일 시점에서 잘못된 접근을 **오류로 검출**할 수 있어 디버깅이 용이하다.

### 캡슐화 vs 정보 은닉 비교

| 구분 | 캡슐화 | 정보 은닉 |
|---|---|---|
| 핵심 개념 | 데이터와 함수를 하나의 단위로 묶음 | 멤버 변수를 외부로부터 감춤 |
| 목적 | 관련 요소를 하나로 응집 | 데이터 보호 및 무결성 유지 |
| 구현 방법 | 클래스(class) 사용 | `private` 접근 지정자 + getter/setter |

> C 언어는 프로그래머가 스스로 조심해야 하지만, C++은 접근 지정자라는 문법 수준의 기능을 통해 이러한 오류를 언어 차원에서 방지한다.
