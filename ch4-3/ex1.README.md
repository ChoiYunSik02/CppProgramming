# 4-3장 실습과제 1번

## 1. 메모리 누수란?

메모리 누수(memory leak)는 프로그램이 동적으로 할당한 메모리를 더 이상 사용하지 않으면서도 해제하지 않아,
해당 메모리를 계속 점유하고 있는 상태를 의미한다.

즉, new로 메모리를 할당한 후 delete를 하지 않으면 발생한다.

---

## 2. 메모리 누수가 왜 심각한 문제인가?

메모리 누수는 다음과 같은 문제를 발생시킨다.

- 프로그램이 사용할 수 있는 메모리가 점점 줄어든다.
- 다른 프로그램이 메모리를 할당받지 못할 수 있다.
- 시스템 전체 성능 저하 발생
- 심한 경우 프로그램이 강제 종료됨 (Out of Memory)

---

## 3. 메모리 누수 방지 방법

- new로 할당한 메모리는 반드시 delete로 해제한다.
- 객체 배열은 delete[] 사용
- 포인터를 잃어버리지 않도록 주의
- 스마트 포인터(C++11 이후) 사용 가능

---

## 결론

메모리 누수는 프로그램 안정성과 성능에 매우 큰 영향을 주므로,
동적 할당 시 반드시 해제까지 신경 써야 한다.
