## 1. 라이브러리의 헤더파일과 목적파일에 들어 있는 내용

| 구성 | 확장자 | 들어 있는 내용 |
|------|--------|----------------|
| **헤더파일** | `.h` | 함수의 선언, 클래스 선언, 상수 정의 등 타입 정보 |
| **목적파일** | `.obj` / `.lib` | 함수의 실제 구현(정의)이 기계어로 컴파일된 코드 |

- 헤더파일은 컴파일러가 타입 정보를 확인하기 위해 사용된다.
- 목적파일은 링크 단계에서 실행파일에 연결된다.

---

## 2. 함수의 선언, 정의, 호출은 각각 어디에 존재하는가?

| 구분 | 위치 | 설명 |
|------|------|------|
| **선언 (Declaration)** | 헤더파일 (`.h`) | 함수의 이름, 반환형, 매개변수 타입을 명시 |
| **정의 (Definition)** | 목적파일 (`.obj`) 또는 소스파일 (`.cpp`) | 함수의 실제 동작 코드 |
| **호출 (Call)** | 함수를 사용하는 소스파일 (`.cpp`) | 선언된 함수를 실제로 사용하는 곳 |

## 3. extern 선언과 변수선언의 차이

**extern 선언**
- 변수가 다른 파일에 이미 정의되어 있음을 컴파일러에게 알리는 것
- **메모리를 할당하지 않는다**

```cpp
extern istream cin;   // cin은 다른 곳에 정의되어 있음을 알림 (메모리 할당 X)
extern ostream cout;  // cout은 다른 곳에 정의되어 있음을 알림 (메모리 할당 X)
```

**변수 선언(정의)**
- 변수를 위한 **실제 메모리를 할당**하는 선언

```cpp
int x = 0;   // 메모리에 x 공간을 할당하고 0으로 초기화
```

### cin, cout의 실제 정의(변수선언)는 어디에 있는가?

`<iostream>` 헤더파일에는 아래와 같이 **extern 선언만** 들어 있다.

```cpp
_EXPORT_STD extern "C++" istream cin;
_EXPORT_STD extern "C++" ostream cout;
```

cin과 cout의 **실제 정의(메모리 할당)** 는 C++ 표준 **라이브러리 목적파일** 안에 존재하며,  
링크 단계에서 해당 목적파일과 연결되어 실제 객체가 사용된다.

## 4. 라이브러리 사용 전에 헤더파일을 include하는 이유

컴파일러는 함수나 객체를 사용할 때 아래 정보를 **미리** 알아야 한다.
- 함수의 반환형
- 매개변수의 개수와 타입
- 클래스의 멤버 구성

헤더파일에는 이러한 **선언(타입 정보)** 이 들어 있으므로,  
`#include`로 헤더파일을 포함시켜야 컴파일러가 올바르게 문법을 검사하고 코드를 생성할 수 있다.

> ❌ include하지 않으면 → 컴파일러가 함수를 찾지 못해 **컴파일 오류** 발생
