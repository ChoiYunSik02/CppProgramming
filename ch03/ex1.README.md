# C++ 구조체(struct)와 클래스(class) 정리

---

## 1. 구조체(struct)와 클래스(class)의 차이점

구조체와 클래스는 C++에서 사용자 정의 자료형을 만드는 두 가지 방법이다.  
문법적으로는 거의 동일하지만, **기본 접근 지정자(default access specifier)** 에서 결정적인 차이가 있다.

| 구분 | `struct` | `class` |
|------|----------|---------|
| 기본 접근 지정자 | `public` | `private` |
| 멤버 변수/함수 | 기본 공개 | 기본 비공개 |
| 상속 시 기본 접근 | `public` 상속 | `private` 상속 |
| 주요 사용 목적 | 단순 데이터 묶음 | 데이터 + 동작(캡슐화) |

### 정리

- `struct`는 멤버가 **기본적으로 public** 이므로 외부에서 직접 접근 가능하다.
- `class`는 멤버가 **기본적으로 private** 이므로 캡슐화(encapsulation)를 기본으로 지원한다.
- C++에서는 접근 지정자만 다를 뿐, 나머지 기능(생성자, 소멸자, 상속, 멤버 함수 등)은 `struct`와 `class` 모두 동일하게 사용할 수 있다.

## 2. 클래스(class)와 객체(object)의 차이점

| 구분 | 클래스 (class) | 객체 (object) |
|------|--------------|--------------|
| 정의 | 데이터와 함수를 묶은 **설계도(틀)** | 클래스로부터 생성된 **실체(인스턴스)** |
| 메모리 | 정의 자체는 메모리를 차지하지 않음 | 생성 시 실제 메모리를 차지함 |
| 비유 | 붕어빵 틀 | 붕어빵 |
| 개수 | 하나의 클래스 정의 | 클래스 하나로 여러 객체 생성 가능 |

### 정리

- **클래스**는 객체를 만들기 위한 **설계도**이며, 선언 자체만으로는 메모리를 할당하지 않는다.
- **객체**는 클래스를 기반으로 실제로 **생성된 실체**이며, 생성 시 메모리가 할당된다.
- 하나의 클래스로 여러 개의 객체를 만들 수 있으며, 각 객체는 서로 독립적인 멤버 변수를 가진다.

## `sizeof(Triangle)` 결과값: **8 (바이트)**
 
### 클래스 메모리 구성 분석
 
`sizeof()` 연산자는 클래스(또는 객체)가 메모리에서 차지하는 **바이트 크기**를 반환한다.  
`Triangle` 클래스의 멤버 구성을 살펴보면 다음과 같다.
 
| 멤버 | 종류 | 타입 | 크기 |
|---|---|---|---|
| `width` | 멤버 변수 | `int` | **4 바이트** |
| `height` | 멤버 변수 | `int` | **4 바이트** |
| `getArea()` | 멤버 함수 | - | **포함 안 됨 (0 바이트)** |
| | | **합계** | **8 바이트** |
 
---
