## Q1. 인자와 매개변수를 구분하여 설명하시오

| 구분 | 위치 | 설명 |
|------|------|------|
| **인자 (Argument)** | 호출하는 함수 (caller) | 함수를 호출할 때 전달하는 값 또는 변수. **호출하는 쪽의 지역변수**이다. |
| **매개변수 (Parameter)** | 호출당하는 함수 (callee) | 인자를 받기 위해 함수 정의부에 선언된 변수. **호출당하는 쪽의 지역변수**이다. |

**예시:**
```cpp
int main() {
    int i = 100;
    sub(i);      // i는 인자 (argument)
}

int sub(int v) { // v는 매개변수 (parameter)
    ...
}
```

> 함수가 호출되면 매개변수가 메모리(스택)에 할당되고, 인자의 값이 매개변수에 복사(초기화)된다.

## Q2. 함수 호출 시 자동으로 처리되는 2가지는 무엇인가?

1. **매개변수(지역변수) 자동 할당** : 함수가 호출되면 스택에 매개변수 공간이 자동으로 생성된다.
2. **인자값으로 매개변수 초기화** : 호출하는 쪽의 인자 값이 매개변수에 자동으로 복사(초기화)된다.

```
호출 전  →  매개변수 할당  →  인자값 복사(초기화)  →  함수 실행
```

## Q3. 값에 의한 호출로 다른 함수의 변수 값을 변경할 수 없는 이유

**핵심 원인: 매개변수는 인자의 "복사본"이기 때문이다.**

- 값에 의한 호출에서는 인자의 **값만** 매개변수에 복사된다.
- 매개변수는 스택에 독립적으로 생성된 **별개의 변수**이다.
- 따라서 함수 내부에서 매개변수를 아무리 변경해도, 원본 인자 변수에는 전혀 영향을 주지 않는다.
- 함수가 종료되면 매개변수는 스택에서 소멸한다.

```cpp
void swap(int a, int b) {
    int tmp = a;
    a = b;
    b = tmp;
    // a, b는 복사본 → main()의 m, n은 변화 없음
}

int main() {
    int m = 2, n = 9;
    swap(m, n);
    cout << m << ' ' << n; // 출력: 2 9 (변경 안 됨)
}
```

## Q4. 주소에 의한 호출을 사용해야 하는 경우 2가지 이상

1. **함수 내부에서 호출자의 변수 값을 직접 변경해야 할 때**
   - 값에 의한 호출은 복사본만 수정하므로, 원본을 수정하려면 주소(포인터)를 전달해야 한다.
   - 예: 두 변수를 교환하는 `swap()` 함수

2. **크기가 큰 객체(또는 구조체)를 전달할 때**
   - 객체 전체를 복사하면 메모리와 시간이 낭비된다.
   - 주소만 전달하면 복사 비용 없이 빠르게 접근할 수 있다.

3. **함수가 여러 값을 동시에 반환해야 할 때**
   - C++에서 함수는 값을 하나만 리턴할 수 있으므로, 여러 결과를 반환하려면 포인터 매개변수를 통해 직접 기록한다.

```cpp
void increase(Circle* p) {
    int r = p->getRadius();
    p->setRadius(r + 1); // 원본 객체 직접 수정
}
```

## Q5. 인자가 객체인 경우 어떤 호출방식이 좋은가?

**→ 주소에 의한 호출(call by address)이 더 좋다.**

**이유:**

| 비교 항목 | 값에 의한 호출 | 주소에 의한 호출 |
|-----------|--------------|----------------|
| 매개변수 | 객체 전체 복사 | 주소(포인터)만 복사 |
| 생성자 호출 | 복사생성자 호출됨 | 호출 안 됨 |
| 소멸자 호출 | 함수 종료 시 소멸자 호출됨 | 호출 안 됨 |
| 메모리 사용 | 객체 크기만큼 추가 사용 | 포인터 크기(4~8바이트)만 사용 |
| 실행 속도 | 느림 | 빠름 |

- 객체는 크기가 크기 때문에 복사 비용이 크다.
- 주소에 의한 호출은 포인터 하나만 전달하므로 **메모리 절약** + **실행속도 향상** 효과가 있다.

## Q6. 함수 반환 과정에서 자동으로 처리되는 2가지

1. **임시변수(Temporary Variable) 자동 생성 및 반환값으로 초기화**
   - 함수가 `return` 할 때, 컴파일러가 자동으로 임시변수를 생성하고 반환값으로 초기화한다.

2. **함수 호출 문장이 임시변수로 대체(치환)**
   - 함수 호출 표현식 자체가 임시변수로 교체되어 대입 연산이 실행된 뒤, 임시변수는 소멸한다.

```
return m;
 → int 임시변수 = m;         ← ① 임시변수 생성 및 초기화
 → result = 임시변수;        ← ② 함수호출문이 임시변수로 대체
 → 임시변수 소멸             ← ③ 대입 후 임시변수 소멸
```

## Q7. 함수 반환 과정에서 임시변수가 필요한 이유

**함수의 지역변수(반환값)는 함수 종료 즉시 스택에서 소멸하기 때문이다.**

- 함수 내부에서 선언된 지역변수는 함수가 끝나면 스택에서 사라진다.
- 만약 임시변수 없이 지역변수를 직접 반환하면, 함수가 종료된 시점에 이미 그 변수는 소멸해서 값을 사용할 수 없다.
- 따라서 **소멸 전에 임시변수에 백업**해 두고, 호출한 쪽에서 임시변수의 값을 안전하게 사용할 수 있도록 하는 것이다.

```
함수 종료
  → 지역변수 소멸 (스택 해제)
  → 임시변수에 미리 저장해 두었으므로 값 보존 가능
  → 대입 완료 후 임시변수도 소멸
```
