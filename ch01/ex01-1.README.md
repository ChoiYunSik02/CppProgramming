## 1. 전처리(선행처리) 단계란?

컴파일 이전에 전처리기(Preprocessor)가 소스코드를 가공하는 단계이다.

| 지시자 | 역할 |
|--------|------|
| `#include` | 지정한 헤더파일의 내용을 해당 위치에 삽입 |
| `#define` | 매크로 상수/함수를 정의하고 소스코드에서 텍스트 치환 |
| `#ifdef` / `#ifndef` | 조건부 컴파일 처리 |
| 주석 제거 | 소스코드 내 모든 주석을 제거 |

전처리가 완료된 후 컴파일 단계로 넘어간다.

## 2. 객체 cin, cout은 어디에 선언되어 있는가?

`cin`과 `cout`은 `<iostream>` 헤더파일에 **extern 변수**로 선언되어 있다.

```cpp
extern istream cin;   // 표준 입력 스트림 객체 선언
extern ostream cout;  // 표준 출력 스트림 객체 선언
```

- **extern 선언** : 변수가 다른 파일에 정의되어 있음을 컴파일러에게 알리는 선언. 메모리를 직접 할당하지 않는다.
- **변수 정의** : 실제 메모리를 할당하는 선언. 예) `int x = 0;`

따라서 `#include <iostream>`을 하면 cin, cout의 선언 정보를 가져오고,  
링크 단계에서 실제 정의(라이브러리 목적파일)와 연결된다.

> ※ Visual Studio에서 cout에 커서를 놓고 **F12(정의로 이동)** 를 누르면  
> `<iostream>` 헤더파일에서 선언 위치를 확인할 수 있다.

## 3. 링크 시 이름 충돌이 발생하는 경우

서로 다른 소스파일에서 **동일한 이름의 전역 함수 또는 전역 변수**를 정의하면  
링크 단계에서 이름 충돌(Name Conflict)이 발생한다.

```cpp
// file1.cpp
void print() { cout << "file1"; }

// file2.cpp
void print() { cout << "file2"; }
// → 링커가 어느 print()를 사용할지 알 수 없어 오류 발생
```

**해결 방법** : C++의 `namespace`(이름공간)를 사용한다.  
`std::cout`처럼 cin, cout은 `std` 네임스페이스 안에 정의되어 있어  
사용자가 만든 변수와 충돌하지 않는다.

## 4. 스트림(Stream)이란?

프로그램과 입출력 장치(키보드, 모니터, 파일 등) 사이에서  
**데이터가 흐르는 추상화된 통로**이다.

| 종류 | 방향 | 객체 |
|------|------|------|
| 입력 스트림 (istream) | 키보드 → 프로그램 | `cin` |
| 출력 스트림 (ostream) | 프로그램 → 모니터 | `cout` |

스트림을 사용하면 실제 장치의 종류에 상관없이 동일한 방식으로 입출력 코드를 작성할 수 있다.

## 5. 표준 출력 스트림을 정의할 때 사용하는 C언어 자료형

**`FILE*`** (FILE 포인터)

C언어에서 표준 출력은 `FILE* stdout` 형태로 정의되어 있으며  
`<stdio.h>` 헤더파일에 선언되어 있다.

---

## 6. 표준 출력 스트림을 정의할 때 사용하는 C++언어 자료형

**`ostream`** 클래스

C++에서 `cout`은 `ostream` 클래스 타입의 전역 객체로 선언되어 있으며  
`<iostream>` 헤더파일의 `std` 네임스페이스 안에 정의되어 있다.  
`<<` 연산자 오버로딩을 통해 다양한 자료형의 출력을 지원한다.
