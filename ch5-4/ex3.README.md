```cpp
#include <iostream>
```


---

```cpp
class MyStack {
```
> 동적 배열 기반의 스택 클래스 선언을 시작한다.

```cpp
    int* p;
```
> 정수를 저장할 동적 배열을 가리키는 포인터다.  
> 이 멤버가 힙 메모리를 가리키기 때문에 깊은 복사 생성자가 반드시 필요하다.

```cpp
    int size;
```
> 할당받은 동적 배열의 크기(최대 저장 가능 개수)를 저장하는 멤버 변수다.

```cpp
    int tos;
```
> "top of stack"의 약자다.  
> 다음에 데이터가 저장될 배열 인덱스를 나타내며, 동시에 현재 스택에 저장된 데이터 개수이기도 하다.  
> 스택이 비어 있을 때 `tos = 0`이다.

```cpp
public:
    MyStack();
```
> 기본 생성자를 선언한다. 아무 인자 없이 빈 스택 객체를 만든다.

```cpp
    MyStack(int size);
```
> 저장 공간의 크기를 인자로 받아 동적 배열을 할당하는 생성자를 선언한다.

```cpp
    MyStack(const MyStack& src);
```
> 복사 생성자를 선언한다.  
> `MyStack b = a;` 또는 `MyStack b(a);` 처럼 객체를 복사할 때 호출된다.  
> `const&`로 받아 원본 객체를 수정하지 않도록 한다.

```cpp
    ~MyStack();
```
> 소멸자를 선언한다. 객체가 소멸될 때 동적 배열을 힙에서 해제한다.

```cpp
    bool push(int n);
```
> 정수 `n`을 스택에 저장하는 함수를 선언한다.  
> 스택이 꽉 차 있으면 `false`, 성공하면 `true`를 반환한다.

```cpp
    bool pop(int& n);
```
> 스택의 탑(top)에서 값을 꺼내 참조 매개변수 `n`에 저장하는 함수를 선언한다.  
> 스택이 비어 있으면 `false`, 성공하면 `true`를 반환한다.  
> 참조(`&`)로 받기 때문에 함수 밖에서도 꺼낸 값을 사용할 수 있다.

```cpp
};
```
> 클래스 선언을 종료한다.

---

```cpp
MyStack::MyStack() : size(0), tos(0), p(nullptr) {}
```
> 기본 생성자 구현이다.  
> 초기화 리스트(`: size(0), tos(0), p(nullptr)`)를 사용해 세 멤버를 모두 초기화한다.  
> `p = nullptr`로 설정해 소멸자에서 `if(p)` 조건이 false가 되어 `delete`를 호출하지 않도록 한다.

---

```cpp
MyStack::MyStack(int size) {
```
> 크기 지정 생성자 구현부를 시작한다.

```cpp
    this->size = size;
```
> 매개변수 `size`를 멤버 변수 `this->size`에 저장한다.  
> 매개변수 이름과 멤버 변수 이름이 같으므로 `this->`로 구분한다.

```cpp
    this->tos  = 0;
```
> 스택이 비어 있는 초기 상태로 tos를 0으로 설정한다.

```cpp
    this->p    = new int[size];
```
> `size` 개수만큼의 `int` 배열을 힙에 동적 할당하고, 그 시작 주소를 `p`에 저장한다.

```cpp
}
```
> 생성자 구현을 종료한다.

---

```cpp
MyStack::MyStack(const MyStack& src) {
```
> 복사 생성자 구현부를 시작한다. `src`는 복사 원본 객체를 참조한다.

```cpp
    this->size = src.size;
```
> 원본의 배열 크기를 그대로 복사한다.

```cpp
    this->tos  = src.tos;
```
> 원본의 현재 스택 상태(몇 개가 쌓여 있는지)를 그대로 복사한다.  
> 이 값이 복사되어야 `b`가 `a`와 동일한 스택 상태로 시작된다.

```cpp
    this->p = new int[src.size];
```
> **깊은 복사의 핵심.**  
> 원본 `src.p`를 그대로 복사하는 것이 아니라, **새로운 독립된 메모리**를 별도로 할당한다.  
> 이로써 `this->p`와 `src.p`는 서로 다른 메모리를 가리키게 된다.

```cpp
    for (int i = 0; i < src.tos; i++) {
```
> 원본에 실제로 저장된 데이터 개수(`src.tos`)만큼 반복한다.  
> `src.size`가 아닌 `src.tos`까지만 복사하면 되는데, 그 이상의 공간은 아직 사용되지 않은 영역이기 때문이다.

```cpp
        this->p[i] = src.p[i];
```
> 원본 배열의 각 원소를 새로 할당한 배열에 하나씩 복사한다.

```cpp
    }
}
```
> 반복문과 복사 생성자 구현을 종료한다.

```cpp
MyStack::~MyStack() {
```
> 소멸자 구현부를 시작한다. 객체가 스코프를 벗어날 때 자동 호출된다.

```cpp
    if (p) delete[] p;
```
> `p`가 `nullptr`이 아닐 때만 동적 배열을 해제한다.  
> `delete[]`를 사용하는 이유는 `new int[]`로 배열을 할당했기 때문이다.  
> 깊은 복사 덕분에 각 객체가 자신만의 `p`를 가지므로 이중 해제 오류가 발생하지 않는다.

```cpp
}
```
> 소멸자 구현을 종료한다.

---

```cpp
bool MyStack::push(int n) {
```
> `push` 함수 구현부를 시작한다.

```cpp
    if (tos == size) return false;
```
> `tos`가 `size`와 같다면 배열이 꽉 찬 상태이므로 `false`를 반환하고 종료한다.

```cpp
    p[tos++] = n;
```
> 현재 `tos` 위치에 `n`을 저장한 뒤 `tos`를 1 증가시킨다.  
> 후위 증가(`tos++`)이므로 먼저 `p[tos]`에 저장하고, 그다음 `tos`가 증가한다.

```cpp
    return true;
}
```
> 저장에 성공했으므로 `true`를 반환하고 함수를 종료한다.

---

```cpp
bool MyStack::pop(int& n) {
```
> `pop` 함수 구현부를 시작한다. `n`을 참조로 받아 꺼낸 값을 전달한다.

```cpp
    if (tos == 0) return false;
```
> `tos`가 0이면 스택이 비어 있으므로 `false`를 반환하고 종료한다.

```cpp
    n = p[--tos];
```
> 전위 감소(`--tos`)로 먼저 `tos`를 1 감소시킨 뒤, `p[tos]`의 값을 `n`에 저장한다.  
> 스택의 가장 위(마지막으로 push된 값)를 꺼내는 LIFO(후입선출) 동작이다.

```cpp
    return true;
}
```
> 꺼내기에 성공했으므로 `true`를 반환하고 함수를 종료한다.

```cpp
int main() {
```
> 프로그램 진입점이다.

```cpp
    MyStack a(10);
```
> 10개의 정수를 저장할 수 있는 스택 `a`를 생성한다.  
> 내부적으로 `new int[10]`이 실행되어 힙에 배열이 할당된다.

```cpp
    a.push(10); a.push(20);
```
> `a`에 10, 20을 순서대로 저장한다. 저장 후 `a.tos = 2`가 된다.  
> 스택 내부 상태: `a.p = [10, 20, ?, ?, ...]`

```cpp
    MyStack b = a;
```
> **복사 생성자 호출.**  
> `MyStack b(a);`와 완전히 동일하다. 선언과 동시에 초기화이므로 대입 연산자가 아닌 복사 생성자가 호출된다.  
> 깊은 복사로 `b`는 `a`와 동일한 데이터를 가지되, 독립된 메모리를 사용한다.  
> 복사 후 `b.p = [10, 20, ...]`, `b.tos = 2`

```cpp
    b.push(30);
```
> `b`에만 30을 추가로 저장한다. `b.tos = 3`이 된다.  
> `a`는 영향을 받지 않는다. (`a.tos`는 여전히 2)

```cpp
    int n;
```
> `pop`으로 꺼낸 값을 저장할 변수를 선언한다.

```cpp
    a.pop(n);
```
> `a`의 탑에서 값을 꺼낸다. `a.tos`가 2 → 1로 감소하고, `n = a.p[1] = 20`이 된다.

```cpp
    cout << "스택 a에서 팝한 값 " << n << endl;
```
> `20`을 출력한다.

```cpp
    b.pop(n);
```
> `b`의 탑에서 값을 꺼낸다. `b.tos`가 3 → 2로 감소하고, `n = b.p[2] = 30`이 된다.

```cpp
    cout << "스택 b에서 팝한 값 " << n << endl;
```
> `30`을 출력한다.

```cpp
    return 0;
}
```
> main 함수를 정상 종료한다.  
> `b`, `a` 순으로 소멸되며 각자의 소멸자가 각자의 `p` 메모리를 해제한다.

## 복사 생성자가 없을 때 (얕은 복사) 발생하는 문제

```
MyStack b = a 실행 후 (얕은 복사 시):

a.p ──┐
      └──▶ [10, 20, ...]  ← 같은 메모리 공유!
b.p ──┘

b 소멸 시 → delete[] b.p → 메모리 해제
a 소멸 시 → delete[] a.p → 이미 해제된 메모리 재해제 → 런타임 오류!
```

깊은 복사를 하면:

```
a.p ──▶ [10, 20, ...]   (a 전용 메모리)
b.p ──▶ [10, 20, ...]   (b 전용 메모리, 별도 할당)

b 소멸: b.p 해제 → a 영향 없음
a 소멸: a.p 해제 → 정상 종료
```
