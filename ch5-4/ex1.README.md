# ch 5-4 실습과제 1 - 복사 생성자 개념 정리

---

## Q1. 일반 생성자와 복사 생성자의 차이는?

| 구분 | 일반 생성자 | 복사 생성자 |
|------|------------|------------|
| 목적 | 새로운 객체를 **초기화**하여 생성 | 기존 객체를 **복사**하여 새 객체 생성 |
| 매개변수 | 초기값을 받는 일반 매개변수 | **같은 클래스 타입의 const 참조** (`const ClassName& src`) |
| 선언 형식 | `ClassName(int x, ...)` | `ClassName(const ClassName& src)` |
| 호출 시점 | `Circle c(30);` 처럼 직접 생성 시 | `Circle b = a;` 또는 `Circle b(a);` 처럼 객체 복사 시 |
| 자동 제공 | 기본 생성자(인수 없음) 자동 제공 | 디폴트 복사 생성자(얕은 복사) 자동 제공 |

### 핵심 차이 요약
- 일반 생성자는 **"무에서 유를 창조"** — 값을 받아 객체의 멤버를 초기화한다.  
- 복사 생성자는 **"유에서 유를 복제"** — 이미 존재하는 객체의 데이터를 새 객체에 그대로 복사한다.
- 동적 메모리를 가진 클래스에서 복사 생성자를 직접 작성하지 않으면 **얕은 복사(shallow copy)** 로 인해 두 객체가 같은 메모리를 공유하는 문제가 발생한다.

## Q2. 대입 연산과 얕은 복사 생성자의 차이는?

| 구분 | 대입 연산 (`=`) | 얕은 복사 생성자 |
|------|---------------|----------------|
| 실행 시점 | **이미 존재하는 객체**에 다른 객체를 대입 | **새 객체를 생성하는 순간** 원본을 복사 |
| 예시 | `Circle c1, c2(30);` → `c1 = c2;` | `Circle c2(30);` → `Circle c1(c2);` |
| 호출 함수 | 대입 연산자 함수 (`operator=`) | 복사 생성자 (`ClassName(const ClassName&)`) |
| 객체 초기화 | 이미 생성된 객체의 값을 **덮어씀** | 생성과 복사가 **동시에** 일어남 |
| 자동 제공 | 디폴트 대입 연산자 자동 삽입 (얕은 복사) | 디폴트 복사 생성자 자동 삽입 (얕은 복사) |

### 공통점
두 경우 모두 **얕은 복사(shallow copy)** 를 기본으로 수행하며,  
멤버 변수가 동적 메모리를 가리키는 경우 **이중 해제(double free)** 문제가 발생할 수 있다.

### 중요한 구별 포인트
```cpp
Circle c2(30);

Circle c1(c2);   // ← 복사 생성자 호출 (객체 생성 시점)
Circle c1 = c2;  // ← 복사 생성자 호출 (선언과 동시에 초기화 → 생성자!)

c1 = c2;         // ← 대입 연산자 호출 (이미 생성된 c1에 대입)
```

> `Circle c1 = c2;` 는 겉보기에 대입처럼 보이지만,  
> **선언과 동시에 초기화**이므로 복사 생성자가 호출된다.

## Q3. 컴파일러가 자동으로 추가하는 멤버 함수 4가지

| 번호 | 자동 추가 함수 | 역할 |
|------|--------------|------|
| ① | **기본 생성자** `ClassName()` | 인수 없이 객체를 생성. 단, 개발자가 다른 생성자를 정의하면 자동 추가 안 됨 |
| ② | **소멸자** `~ClassName()` | 객체 소멸 시 호출. 개발자가 정의하지 않으면 아무 동작도 하지 않는 소멸자 자동 삽입 |
| ③ | **복사 생성자** `ClassName(const ClassName& src)` | 객체 복사 시 호출. **얕은 복사** 수행. 동적 메모리가 있으면 직접 정의 필요 |
| ④ | **디폴트 대입 연산자** `ClassName& operator=(const ClassName& src)` | `=` 연산으로 객체 대입 시 호출. 마찬가지로 **얕은 복사** 수행 |
