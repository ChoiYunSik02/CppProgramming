# ch5-2 실습과제 1 — 참조에 의한 호출 vs 주소에 의한 호출

---

## 문제 1. 참조에 의한 호출과 주소에 의한 호출의 차이점을 설명하라.

### 값에 의한 호출 (Call by Value)

함수 호출 시 인자의 **값을 복사**하여 매개변수에 전달한다.  
매개변수는 별도의 메모리 공간을 가지므로, 함수 내부에서 매개변수를 변경해도 **원본 변수에 영향을 미치지 않는다.**

```cpp
void modify(int value) {
    value = 99;  // 복사본만 변경, 원본 number에는 영향 없음
}
int main() {
    int number = 1;
    modify(number);
    cout << number;  // 출력: 1 (원본 그대로)
}
```

---

### 주소에 의한 호출 (Call by Address / Pointer)

함수 호출 시 인자 변수의 **메모리 주소**를 포인터 매개변수로 전달한다.  
함수 내부에서 포인터를 통해 원본 변수에 직접 접근하므로 원본 값을 변경할 수 있다.  
단, `&`(주소 연산자)와 `*`(간접 참조 연산자)를 명시적으로 사용해야 한다.

```cpp
void modify(int* value) {
    *value = 99;  // 포인터로 원본 직접 접근
}
int main() {
    int number = 1;
    modify(&number);   // 주소를 전달
    cout << number;    // 출력: 99
}
```

---

### 참조에 의한 호출 (Call by Reference)

함수의 매개변수를 **참조(Reference)** 로 선언한다.  
함수 호출 시 참조 매개변수는 이름만 생기고 별도의 메모리 공간은 할당되지 않는다.  
인자 변수가 원본이 되고, 참조 매개변수는 그 **별명(alias)** 이 된다.  
따라서 함수 내부에서 참조 매개변수에 대한 연산은 **원본 변수에 대한 연산과 동일**하다.

```cpp
void modify(int& value) {
    value = 99;  // value는 number의 별명 → 원본 변경
}
int main() {
    int number = 1;
    modify(number);    // 값에 의한 호출처럼 자연스럽게 전달
    cout << number;    // 출력: 99
}
```

---

### 비교 요약표

| 구분 | 값에 의한 호출 | 주소에 의한 호출 | 참조에 의한 호출 |
|------|--------------|----------------|----------------|
| 전달 방식 | 값 복사 | 주소 전달 | 별명(참조) 생성 |
| 원본 변경 | ❌ 불가 | ✅ 가능 | ✅ 가능 |
| 별도 메모리 할당 | ✅ 할당됨 | ✅ 포인터 공간 할당 | ❌ 할당 안 됨 |
| 호출 문법 | `modify(n)` | `modify(&n)` | `modify(n)` |
| `&`, `*` 연산자 필요 | ❌ | ✅ | ❌ |

## 문제 2. 참조에 의한 호출의 장점은 무엇인가?

### ① 원본 변수를 직접 수정할 수 있다

주소에 의한 호출과 동일한 효과를 가진다.  
함수 내부에서 참조 매개변수를 수정하면 호출한 쪽의 원본 변수가 변경된다.

### ② 사용법이 간결하다

주소에 의한 호출과 달리 `&`(주소 연산자), `*`(간접 참조 연산자)를 사용할 필요가 없다.  
**호출하는 문법은 값에 의한 호출과 동일하지만, 효과는 주소에 의한 호출과 동일**하다.

```cpp
// 주소에 의한 호출 — 연산자 필요
void swap(int* a, int* b) { int t = *a; *a = *b; *b = t; }
swap(&m, &n);   // & 필요

// 참조에 의한 호출 — 깔끔하고 직관적
void swap(int& a, int& b) { int t = a; a = b; b = t; }
swap(m, n);     // & 불필요
```

### ③ 불필요한 복사 비용이 없다

값에 의한 호출은 객체나 큰 구조체를 전달할 때 복사 비용이 크다.  
참조에 의한 호출은 복사 없이 원본에 바로 접근하므로 **성능상 유리**하다.

### ④ 여러 값을 동시에 반환할 수 있다

`return`문은 하나의 값만 반환할 수 있지만, 참조 매개변수를 활용하면 여러 결과를 호출자에게 전달할 수 있다.

```cpp
// avg는 참조로 전달 → 함수 내부에서 평균값 저장
// 리턴값으로는 성공/실패 여부 전달
bool average(int a[], int size, int& avg);
```

---

> **결론:** 주소에 의한 호출이 필요한 경우, 참조에 의한 호출을 사용하는 것이 더 간결하고 안전하다.
