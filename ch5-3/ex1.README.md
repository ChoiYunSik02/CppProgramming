# Ch05-3 실습과제1 - 참조에 의한 호출 & 참조 리턴 이론 정리

## 1. 참조에 의한 호출의 장점 (메모리 사용량 / 처리 시간)

### 메모리 사용량 측면

값에 의한 호출(Call by Value)은 함수 호출 시 인자의 **복사본**을 위한 새로운 메모리 공간이 스택에 할당된다.
특히 객체를 전달할 때 객체 전체가 복사되므로, 객체의 크기가 클수록 추가 메모리 낭비가 크다.

반면 참조에 의한 호출(Call by Reference)은 **별명(참조)만 생성**되며 실제 공간이 새로 할당되지 않는다.
따라서 객체가 아무리 커도 메모리 추가 사용이 없다.

```
// 값에 의한 호출 - 복사본 공간 생성
void func(Circle c) { ... }   // Circle 객체 전체 복사 → 메모리 낭비

// 참조에 의한 호출 - 공간 없음
void func(Circle& c) { ... }  // 별명만 생성 → 추가 메모리 없음
```

### 처리 시간 측면

값에 의한 호출은 복사 과정에서 **복사 생성자(Copy Constructor)** 가 호출되고,
함수 종료 시 **소멸자(Destructor)** 도 호출된다. 이 과정이 처리 시간을 소비한다.

참조에 의한 호출은 생성자와 소멸자가 **전혀 호출되지 않으므로** 처리 시간이 훨씬 빠르다.

| 구분 | 값에 의한 호출 | 참조에 의한 호출 |
|------|--------------|----------------|
| 메모리 | 복사본 공간 할당 (객체 크기만큼) | 공간 할당 없음 |
| 생성자 | 복사 생성자 호출 | 호출 안 됨 |
| 소멸자 | 소멸자 호출 | 호출 안 됨 |
| 처리속도 | 상대적으로 느림 | 빠름 |

## 2. C++ 함수의 반환 과정 상세 설명

### 기본 타입 값 반환

함수가 `int`, `double` 등 기본 타입 값을 리턴할 때:
1. 리턴할 값을 저장하는 **이름 없는 임시 변수(temporary variable)** 가 생성된다.
2. 그 임시 변수에 리턴값이 복사된다.
3. 호출한 쪽에서 그 임시 변수의 값을 읽어 대입한다.
4. 임시 변수는 소멸된다.
5. 리턴값은 **상수(rvalue)** 로 취급되므로 좌변에 올 수 없다.

```cpp
int simplefn(int n) {
    int m = n * 2;
    return m;       // int 임시변수 = m → result = 임시변수
}
int main() {
    int result = simplefn(5); // result = 10
    simplefn(5) = 10;         // 컴파일 에러! (상수에 대입 불가)
}
```

### 객체(클래스 타입) 반환

함수가 객체를 리턴할 때:
1. 리턴할 객체의 **임시 객체(temporary object)** 가 생성된다. (복사 생성자 호출)
2. 임시 객체가 호출한 쪽의 변수에 대입된다. (대입 연산자 호출)
3. 임시 객체는 소멸된다. (소멸자 호출)

→ 복사 생성자, 대입 연산자, 소멸자가 추가로 호출되어 **오버헤드** 발생

```cpp
Circle simplefn(Circle n) {
    Circle m;
    return m;     // Circle 임시객체 = m (복사생성자), result = 임시객체 (대입연산자)
}
```

## 3. 반환형이 참조로 선언된 경우 반환값의 의미

함수의 반환형을 `T&` (참조)로 선언하면, 함수는 값(복사본)이 아니라
**기존에 존재하는 변수에 대한 별명(참조)** 을 리턴한다.

### 핵심 차이

| 구분 | 값 리턴 (`T`) | 참조 리턴 (`T&`) |
|------|------------|----------------|
| 리턴 대상 | 복사된 임시 값 | 기존 변수의 별명 |
| 좌변 사용 | 불가 (상수 취급) | 가능 (변수 취급) |
| 임시 공간 | 임시 변수/객체 생성 | 공간 할당 없음 |

### 예시

```cpp
char c = 'a';

// 값 리턴 → 상수 취급
char get() { return c; }
char a = get();   // a = 'a' (OK)
get() = 'b';      // 컴파일 에러! 좌변에 상수 불가

// 참조 리턴 → 변수(공간) 취급
char& find() { return c; }  // c의 별명을 리턴
char a = find();    // a = 'a' (OK)
find() = 'b';       // c = 'b' (OK! 좌변에 변수 가능)
char& ref = find(); // ref는 c의 별명
ref = 'M';          // c = 'M'
```

### 주의사항: 지역변수의 참조 리턴 금지

```cpp
char& find() {
    char x = 'a';
    return x;   // ❌ 위험! x는 함수 종료 후 소멸 → 존재하지 않는 변수 참조
}

char c = 'a';   // 전역변수
char& find() {
    return c;   // ✅ 안전. 전역변수는 함수 종료 후에도 존재
}
```

참조 리턴 시 반드시 **전역변수, 동적 할당 변수, 또는 외부에서 전달받은 참조** 만 리턴해야 한다.

## 4. 참조 리턴 응용 예제 - 메서드 체이닝 (Method Chaining)

참조 리턴의 대표적인 활용: `return *this`를 이용한 **연속 호출(체이닝)**

```cpp
#include <iostream>
using namespace std;

class Circle {
    int radius;
public:
    Circle(int r) : radius(r) {}

    // 참조 리턴 → *this를 리턴하여 체이닝 가능
    Circle& plus(int n) {
        radius += n;
        return *this;
    }

    Circle& minus(int n) {
        radius -= n;
        return *this;
    }

    int getRadius() { return radius; }
};

int main() {
    Circle a(5);
    a.plus(1).plus(2).plus(3);  // (5+1+2+3) = 11, 모두 원본 a에 반영
    cout << "반지름: " << a.getRadius() << endl; // 11

    return 0;
}
```

**왜 가능한가?**
- `plus()`가 `Circle&`(참조)를 리턴하므로 `.plus(2)`는 복사본이 아닌 **원본 a** 에 대해 호출됨
- 만약 `Circle plus()`(값 리턴)이면 `.plus(2)`는 **임시 복사본** 에 대해 호출되어 a에 반영 안 됨

**참조 리턴 vs 값 리턴 결과 비교**

```
참조 리턴: a(5) → plus(1) → a(6) → plus(2) → a(8) → plus(3) → a(11)  ✅
값   리턴: a(5) → plus(1) → a(6) → 복사본(6) → plus(2) → 복사본(8)...  a는 6에서 멈춤 ❌
```
