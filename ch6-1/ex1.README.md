## Chapter6-1 실습과제 1 — C언어와 C++언어의 함수 구분 방식 차이
 
### 📌 C언어의 함수 구분 방식
 
C언어에서는 **함수 이름(name)만으로** 함수를 구분한다.
따라서 같은 이름의 함수를 두 개 이상 정의하면 **컴파일 오류**가 발생한다.
 
```c
// C언어 — 동일 이름 함수 2개 → 컴파일 오류
int sum(int a, int b) { return a + b; }
double sum(double a, double b) { return a + b; }  // ❌ 오류: 이름 중복
```
 
> C 컴파일러는 내부적으로 함수를 `sum` 이라는 이름 하나로만 기억하므로,
> 같은 이름이 두 번 등장하면 어떤 함수를 호출해야 할지 알 수 없다.
 
---
 
### 📌 C++언어의 함수 구분 방식
 
C++에서는 **함수 이름 + 매개변수의 타입/개수**를 함께 사용하여 함수를 구분한다.
이를 **이름 맹글링(Name Mangling)** 이라고 하며, 컴파일러가 내부적으로
`sum_int_int`, `sum_double_double` 처럼 함수를 고유하게 변환하여 저장한다.
 
```cpp
// C++ — 매개변수가 다르면 같은 이름도 허용 (함수 중복)
int sum(int a, int b) { return a + b; }           // ✅ int 버전
double sum(double a, double b) { return a + b; }  // ✅ double 버전
```
 
---
 
### 📊 비교 요약표
 
| 구분 | C언어 | C++언어 |
|------|-------|---------|
| 함수 구분 기준 | **함수 이름만** | **함수 이름 + 매개변수 타입/개수** |
| 동일 이름 함수 허용 | ❌ 불가 | ✅ 가능 (함수 중복) |
| 내부 처리 방식 | 이름 그대로 저장 | Name Mangling으로 고유 이름 생성 |
| 리턴 타입으로 구분 | ❌ 불가 | ❌ 불가 (매개변수만 기준) |
 
> 핵심 요약: **C는 이름만으로**, **C++는 이름과 매개변수로** 함수를 구분한다.
> 리턴 타입(반환형)은 C, C++ 모두 함수 구분 기준에 포함되지 않는다.
